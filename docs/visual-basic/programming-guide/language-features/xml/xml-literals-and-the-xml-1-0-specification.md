# <a name="xml-literals-and-the-xml-10-specification-visual-basic"></a>Literales XML y la especificación XML 1.0 (Visual Basic)
La sintaxis de literales XML en [!INCLUDE[vbprvb](../../../../csharp/programming-guide/concepts/linq/includes/vbprvb_md.md)] admite la mayor parte de la especificación de lenguaje de marcado Extensible (XML) 1.0. Para obtener más información acerca de la especificación XML 1.0, consulte [Extensible Markup Language (XML) 1.0](http://go.microsoft.com/fwlink/?LinkId=73927) en el sitio Web de W3C.  
  
## <a name="what-visual-basic-does-not-support"></a>¿Qué no admite Visual Basic  
  
-   Un literal XML no puede contener una definición de tipo de documento (DTD).  
  
-   Un literal de documento XML debe empezar con una declaración de documento XML.  
  
-   Un literal XML no puede contener más de 65.535 caracteres en una sola línea.  
  
-   Los prefijos de espacio de nombres XML, nombres de elemento y nombres de atributo no pueden contener más de 1.024 caracteres.  
  
## <a name="extra-features-that-visual-basic-supports"></a>Características adicionales que admite Visual Basic  
  
-   La sintaxis de expresiones incrustadas permitida en los literales de documento y elemento no es XML válido.
