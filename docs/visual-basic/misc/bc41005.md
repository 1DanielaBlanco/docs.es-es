# <a name="operator-without-an-39as39-clause-type-of-object-assumed"></a>Operador sin una &#39; Como &#39; cláusula; tipo de objeto supone
Un procedimiento de operador no especifica una cláusula `As` .  
  
 Una cláusula `As` identifica un tipo de datos que se debe asociar a un elemento de programación. En un elemento [Operator Statement](../../visual-basic/language-reference/statements/operator-statement.md), especifica el tipo de datos del valor que el procedimiento del operador devuelve al código de llamada. Si no incluye una cláusula `As` en la instrucción `Operator` , el tipo de datos devuelto es `Object`de forma predeterminada.  
  
 De forma predeterminada, este mensaje es una advertencia. Para obtener información sobre cómo ocultar las advertencias o cómo tratarlas como errores, vea [Configuring Warnings in Visual Basic](/visualstudio/ide/configuring-warnings-in-visual-basic).  
  
 **Identificador de error:** BC41005  
  
## <a name="to-correct-this-error"></a>Para corregir este error  
  
-   Incluya una cláusula `As` en la instrucción `Operator` para especificar el tipo de datos devuelto.
