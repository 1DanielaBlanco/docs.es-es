<span data-ttu-id="f1be4-103">Una clase utiliza un elemento [Inherits Statement](../../visual-basic/language-reference/statements/inherits-statement.md) para especificar una clase base, pero no puede acceder a ningún constructor de la clase base deseada.</span><span class="sxs-lookup"><span data-stu-id="f1be4-103">A class uses an [Inherits Statement](../../visual-basic/language-reference/statements/inherits-statement.md) to specify a base class, but it cannot access any constructor on the intended base class.</span></span>  
  
 <span data-ttu-id="f1be4-104">Esto puede ocurrir si la clase base deseada no tiene constructores o si tiene constructores con niveles de acceso que impiden el acceso desde otra clase.</span><span class="sxs-lookup"><span data-stu-id="f1be4-104">This can happen if the intended base class has no constructors or if it has constructors with access levels that prevent access from another class.</span></span>  
  
 <span data-ttu-id="f1be4-105">Cuando se hereda de una clase, su constructor debería llamar el constructor de clase base con [MyBase](~/docs/visual-basic/programming-guide/program-structure/me-my-mybase-and-myclass.md#mybase).</span><span class="sxs-lookup"><span data-stu-id="f1be4-105">When you inherit a class, your constructor should call the base class constructor using [MyBase](~/docs/visual-basic/programming-guide/program-structure/me-my-mybase-and-myclass.md#mybase).</span></span> <span data-ttu-id="f1be4-106">Si no realiza esta llamada o ni siquiera escribe un constructor explícito, Visual Basic genera un constructor implícito que llama a `MyBase.New()`.</span><span class="sxs-lookup"><span data-stu-id="f1be4-106">If you do not make this call, or if you do not even write an explicit constructor, Visual Basic generates an implicit constructor that calls `MyBase.New()`.</span></span>  
  
 <span data-ttu-id="f1be4-107">**Id. de error:** BC31399</span><span class="sxs-lookup"><span data-stu-id="f1be4-107">**Error ID:** BC31399</span></span>
