# <a name="class-ltclassname1gt-must-declare-a-sub-new-because-its-base-class-ltclassname2gt-has-more-than-one-accessible-sub-new-that-can-be-called-with-no-arguments"></a><span data-ttu-id="1d32c-102">La clase&lt;nombredeclase1&gt;' debe declarar un 'Sub New' porque su clase base&lt;nombreclase2&gt;' tiene más de un 'Sub New' accesible que se pueda llamar sin argumentos</span><span class="sxs-lookup"><span data-stu-id="1d32c-102">Class '&lt;classname1&gt;' must declare a 'Sub New' because its base class '&lt;classname2&gt;' has more than one accessible 'Sub New' that can be called with no arguments</span></span>

 <span data-ttu-id="1d32c-104">Cuando una clase derivada no declara un constructor, Visual Basic intenta generar un constructor sin parámetros implícito que llama a `MyBase.New()`.</span><span class="sxs-lookup"><span data-stu-id="1d32c-104">When a derived class does not declare a constructor, Visual Basic attempts to generate an implicit parameterless constructor that calls `MyBase.New()`.</span></span> <span data-ttu-id="1d32c-105">Si no hay ningún constructor accesible en la clase base que se pueda llamar sin argumentos, o si hay más de uno, Visual Basic no se puede generar un constructor implícito.</span><span class="sxs-lookup"><span data-stu-id="1d32c-105">If there is no accessible constructor in the base class that can be called without arguments, or if there is more than one, Visual Basic cannot generate an implicit constructor.</span></span>
