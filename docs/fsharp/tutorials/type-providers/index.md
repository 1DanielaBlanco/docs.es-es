providers are a significant part of F# 3.0 support for information-rich programming.</span></span> <span data-ttu-id="1c955-109">La clave para este tipo de programación es eliminar las barreras para information-rich programming is to eliminate barriers to working with diverse information sources found on the Internet and in modern enterprise environments.</span></span> <span data-ttu-id="1c955-110">Una barrera importante para incluir una fuente de información en un programa es la necesidad de representar dicha información como tipos, propiedades y métodos para usarlos en un entorno de lenguaje de programación.</span><span class="sxs-lookup"><span data-stu-id="1c955-110">One significant barrier to including a source of information into a program is the need to represent that information as types, properties, and methods for use in a programming language environment.</span></span> <span data-ttu-id="1c955-111">La escritura manual de estos tipos lleva mucho Sin embargo, los tipos convencionales de generación de código no se integran bien en los modos exploratorios de programación admitidos por F#, puesto que el código generado debe reemplazarse cada vez que se ajuste una referencia de servicio.</span><span class="sxs-lookup"><span data-stu-id="1c955-112">A common alternative is to use a code generator which adds files to your project; however, the conventional types of code generation do not integrate well into exploratory modes of programming supported by F# because the generated code must be replaced each time a service reference is adjusted.</span></span>

<span data-ttu-id="1c955-113">Los tipos proporcionados por los proveedores de tipo de F# normalmente se basan en fuentes de información externas.</span><span class="sxs-lookup"><span data-stu-id="1c955-113">The types provided by F# type providers are usually based on external information sources.</span></span> <span data-ttu-id="1c955-114">Por ejemplo, un proveedor de tipo de F# para SQL proporcionará los tipos, propiedades y métodos necesarios para trabajar directamente con las tablas de cualquier base de datos SQL a la que se tenga acceso.</span><span class="sxs-lookup"><span data-stu-id="1c955-114">For example, an F# type provider for SQL will provide the types, properties, and methods you need to work directly with the tables of any SQL database you have access to.</span></span> <span data-ttu-id="1c955-115">De igual forma, un proveedor de tipo para los servicios Web WSDL proporcionará los tipos, propiedades y métodos necesarios para trabajar directamente con cualquier servicio Web WSDL.</span><span class="sxs-lookup"><span data-stu-id="1c955-115">Similarly, a type provider for WSDL web services will provide the types, properties, and methods you need to work directly with any WSDL web service.</span></span>

<span data-ttu-id="1c955-116">El conjunto de tipos, propiedades y métodos proporcionados por un proveedor de tipo de F# puede depender de los parámetros especificados en el código del programa.</span><span class="sxs-lookup"><span data-stu-id="1c955-116">The set of types, properties, and methods provided by an F# type provider can depend on parameters given in program code.</span></span> <span data-ttu-id="1c955-117">Por ejemplo, un proveedor de tipo puede proporcionar diferentes tipos en función de una cadena de conexión o de una dirección URL de servicio.</span><span class="sxs-lookup"><span data-stu-id="1c955-117">For example, a type provider can provide different types depending on a connection string or a service URL.</span></span> <span data-ttu-id="1c955-118">De este modo, el espacio de información disponible por medio de una cadena de conexión o dirección URL se integra directamente en el programa.</span><span class="sxs-lookup"><span data-stu-id="1c955-118">In this way, the information space available by means of a connection string or URL is directly integrated into your program.</span></span> <span data-ttu-id="1c955-119">Un proveedor de tipo también puede asegurar que solo se expandan grupos de tipos a petición, es decir, que se expandan si el programa hace realmente referencia a los tipos.</span><span class="sxs-lookup"><span data-stu-id="1c955-119">A type provider can also ensure that groups of types are only expanded on demand; that is, they are expanded if the types are actually referenced by your program.</span></span> <span data-ttu-id="1c955-120">Esto permite la integración directa a petición de espacios de información a gran escala, tales como mercados de datos en línea de forma fuertemente tipada.</span><span class="sxs-lookup"><span data-stu-id="1c955-120">This allows for the direct, on-demand integration of large-scale information spaces such as online data markets in a strongly typed way.</span></span>

<span data-ttu-id="1c955-121">F# contiene varios proveedores de tipo integrados para servicios de datos empresariales y de Internet de uso habitual.</span><span class="sxs-lookup"><span data-stu-id="1c955-121">F# contains several built-in type providers for commonly used Internet and enterprise data services.</span></span> <span data-ttu-id="1c955-122">Estos proveedores de tipo proporcionan acceso simple y regular a bases de datos relacionales SQL y a servicios basados en red de tipo OData y WSDL y admiten el uso de consultas LINQ de F# en estos orígenes de datos.</span><span class="sxs-lookup"><span data-stu-id="1c955-122">These type providers give simple and regular access to SQL relational databases and network-based OData and WSDL services and support the use of F# LINQ queries against these data sources.</span></span>

<span data-ttu-id="1c955-123">En caso necesario, se pueden crear proveedores de tipo personalizados propios o se puede hacer referencia a proveedores de tipo creados por otros.</span><span class="sxs-lookup"><span data-stu-id="1c955-123">Where necessary, you can create your own custom type providers, or reference type providers that have been created by others.</span></span> <span data-ttu-id="1c955-124">Por ejemplo, suponga que su organización tiene un servicio de datos que proporciona un número elevado y creciente de conjuntos de datos con nombre, cada uno con su propio esquema de datos estable.</span><span class="sxs-lookup"><span data-stu-id="1c955-124">For example, assume your organization has a data service providing a large and growing number of named data sets, each with its own stable data schema.</span></span> <span data-ttu-id="1c955-125">Se puede elegir crear un proveedor de tipo que lea los esquemas y presente los últimos conjuntos de datos disponibles al programador de forma fuertemente tipada.</span><span class="sxs-lookup"><span data-stu-id="1c955-125">You may choose to create a type provider that reads the schemas and presents the latest available data sets to the programmer in a strongly typed way.</span></span>
