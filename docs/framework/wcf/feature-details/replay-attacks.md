# <a name="replay-attacks"></a><span data-ttu-id="55320-102">Ataques por repetición</span><span class="sxs-lookup"><span data-stu-id="55320-102">Replay Attacks</span></span>
<span data-ttu-id="55320-103">A *ataque de reproducción* se produce cuando un atacante copia una secuencia de mensajes entre dos partes y reproduce la secuencia a una o varias de las partes.</span><span class="sxs-lookup"><span data-stu-id="55320-103">A *replay attack* occurs when an attacker copies a stream of messages between two parties and replays the stream to one or more of the parties.</span></span> <span data-ttu-id="55320-104">A menos que se mitigue, el equipo objeto del ataque procesa la secuencia como mensajes legítimos, produciendo una gama de malas consecuencias, como pedidos redundantes de un elemento.</span><span class="sxs-lookup"><span data-stu-id="55320-104">Unless mitigated, the computers subject to the attack process the stream as legitimate messages, resulting in a range of bad consequences, such as redundant orders of an item.</span></span>  
  
## <a name="bindings-may-be-subject-to-reflection-attacks"></a><span data-ttu-id="55320-105">Los enlaces pueden estar sujetos a ataques de reflexión</span><span class="sxs-lookup"><span data-stu-id="55320-105">Bindings May Be Subject to Reflection Attacks</span></span>  
 <span data-ttu-id="55320-106">*Ataques de reflexión* son repeticiones de mensajes devueltos a un remitente como si procedieran del receptor como la respuesta.</span><span class="sxs-lookup"><span data-stu-id="55320-106">*Reflection attacks* are replays of messages back to a sender as if they came from the receiver as the reply.</span></span> <span data-ttu-id="55320-107">El estándar *la detección de reproducción* en el [!INCLUDE[indigo1](../../../../includes/indigo1-md.md)] mecanismo no controlan automáticamente esto.</span><span class="sxs-lookup"><span data-stu-id="55320-107">The standard *replay detection* in the [!INCLUDE[indigo1](../../../../includes/indigo1-md.md)] mechanism does not automatically handle this.</span></span>  
  
 <span data-ttu-id="55320-108">Los ataques de reflexión se mitigan de forma predeterminada porque el modelo del servicio de [!INCLUDE[indigo2](../../../../includes/indigo2-md.md)] agrega un Id. de mensaje firmado a los mensajes de solicitud y espera un encabezado `relates-to` firmado en los mensajes de respuesta.</span><span class="sxs-lookup"><span data-stu-id="55320-108">Reflection attacks are mitigated by default because the [!INCLUDE[indigo2](../../../../includes/indigo2-md.md)] service model adds a signed message ID to request messages and expects a signed `relates-to` header on response messages.</span></span> <span data-ttu-id="55320-109">Por consiguiente, el mensaje de solicitud no se puede volver a reproducir como una respuesta.</span><span class="sxs-lookup"><span data-stu-id="55320-109">Consequently, the request message cannot be replayed as a response.</span></span> <span data-ttu-id="55320-110">En los escenarios de mensaje confiables (RM) seguros, los ataques de reflexión se mitigan porque:</span><span class="sxs-lookup"><span data-stu-id="55320-110">In secure reliable message (RM) scenarios, reflection attacks are mitigated because:</span></span>  
  
-   <span data-ttu-id="55320-111">Los esquemas de la secuencia de creación y los del mensaje de respuesta de la secuencia de creación son diferentes.</span><span class="sxs-lookup"><span data-stu-id="55320-111">The create sequence and create sequence response message schemas are different.</span></span>  
  
-   <span data-ttu-id="55320-112">Para las secuencias símplex, los mensajes de secuencias que envía el cliente no se pueden volver a reproducir porque el cliente no puede entender tales mensajes.</span><span class="sxs-lookup"><span data-stu-id="55320-112">For simplex sequences, sequence messages the client sends cannot be replayed back to it because the client cannot understand such messages.</span></span>  
  
-   <span data-ttu-id="55320-113">Para las secuencias dúplex, los dos Id. de secuencia deben ser únicos.</span><span class="sxs-lookup"><span data-stu-id="55320-113">For duplex sequences, the two sequence IDs must be unique.</span></span> <span data-ttu-id="55320-114">Así, un mensaje de secuencia saliente no se puede volver a reproducir como un mensaje de secuencia de entrada (todos los encabezados de secuencia y cuerpos también se firman).</span><span class="sxs-lookup"><span data-stu-id="55320-114">Thus, an outgoing sequence message cannot be replayed back as an incoming sequence message (all sequence headers and bodies are signed, too).</span></span>  
  
 <span data-ttu-id="55320-115">Los únicos enlaces que son susceptibles a los ataques de reflexión son aquellos sin WS-Addressing: los enlaces personalizados que tienen WS-Addressing deshabilitado y utilizan la seguridad basada en clave simétrica.</span><span class="sxs-lookup"><span data-stu-id="55320-115">The only bindings that are susceptible to reflection attacks are those without WS-Addressing: custom bindings that have WS-Addressing disabled and use the symmetric key-based security.</span></span> <span data-ttu-id="55320-116">El <xref:System.ServiceModel.BasicHttpBinding> no utiliza WS-Addressing de forma predeterminada, pero no utiliza la seguridad basada en clave simétrica de tal modo que puede ser vulnerable a este ataque.</span><span class="sxs-lookup"><span data-stu-id="55320-116">The <xref:System.ServiceModel.BasicHttpBinding> does not use WS-Addressing by default, but it does not use symmetric key-based security in a way that allows it to be vulnerable to this attack.</span></span>  
  
 <span data-ttu-id="55320-117">La mitigación para los enlaces personalizados consiste no establecer el contexto de seguridad o requerir encabezados WS-Addressing.</span><span class="sxs-lookup"><span data-stu-id="55320-117">The mitigation for custom bindings is to not establish security context or to require WS-Addressing headers.</span></span>

 <span data-ttu-id="55320-124">Las mitigaciones incluyen:</span><span class="sxs-lookup"><span data-stu-id="55320-124">Mitigations include:</span></span>

-   <span data-ttu-id="55320-127">Configure el servicio para utilizar la seguridad de nivel de transporte.</span><span class="sxs-lookup"><span data-stu-id="55320-127">Configure the service to use transport-level security.</span></span>
