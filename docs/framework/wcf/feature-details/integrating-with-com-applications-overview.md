> [!NOTE]
>  <span data-ttu-id="4c391-106">El moniker de servicio utiliza un canal de comunicación de WCF para todas las comunicaciones.</span><span class="sxs-lookup"><span data-stu-id="4c391-106">The service moniker uses a WCF communication channel for all communication.</span></span> <span data-ttu-id="4c391-107">Los mecanismos de identidad y seguridad para ese canal difieren de aquéllos utilizados en los proxies COM y DCOM estándar.</span><span class="sxs-lookup"><span data-stu-id="4c391-107">The security and identity mechanisms for that channel differ from those used in standard COM and DCOM proxies.</span></span> <span data-ttu-id="4c391-108">Además, dado que el moniker de servicio utiliza un canal de comunicación de WCF, el período de tiempo de espera predeterminado es un minuto para todas las llamadas.</span><span class="sxs-lookup"><span data-stu-id="4c391-108">In addition, because the service moniker uses a WCF communication channel the default timeout period is one minute for all calls.</span></span>  
  
 <span data-ttu-id="4c391-109">El moniker de servicio se utiliza con la `GetObject` función para proporcionar a los programadores no administrados con un enfoque fuertemente tipado, específico de COM para llamar a servicios Web de WCF.</span><span class="sxs-lookup"><span data-stu-id="4c391-109">The service moniker is used with the `GetObject` function to provide the unmanaged developer with a strongly-typed, COM-specific approach for calling WCF Web services.</span></span> <span data-ttu-id="4c391-110">Esto requiere un local, visible para COM definición del contrato del servicio Web de WCF y el enlace que se va a usar.</span><span class="sxs-lookup"><span data-stu-id="4c391-110">This requires a local, COM-visible definition of the WCF Web service contract and the binding that is to be used.</span></span> <span data-ttu-id="4c391-111">Al igual que otros clientes WCF, el moniker de servicio debe construir un canal con tipo para el servicio, aunque esta construcción del canal se produce de forma transparente para el programador COM en la primera llamada de método.</span><span class="sxs-lookup"><span data-stu-id="4c391-111">Like other WCF clients, the service moniker must construct a typed channel to the service, though this channel construction occurs transparently to the COM programmer on the first method call.</span></span>  
  
 <span data-ttu-id="4c391-112">Al igual que otros clientes WCF, al utilizar el moniker, las aplicaciones especificar la dirección, enlace y contrato para comunicarse con un servicio.</span><span class="sxs-lookup"><span data-stu-id="4c391-112">In common with other WCF clients, when using the moniker, applications specify the address, binding and contract to communicate with a service.</span></span> <span data-ttu-id="4c391-113">El contrato se puede especificar de una de las maneras siguientes:</span><span class="sxs-lookup"><span data-stu-id="4c391-113">The contract can be specified in one of the following ways:</span></span>

-   <span data-ttu-id="4c391-116">Contrato de MEX: el contrato se recupera en el tiempo de ejecución a partir de un extremo de intercambio de metadatos (MEX).</span><span class="sxs-lookup"><span data-stu-id="4c391-116">MEX contract – the contract is retrieved at runtime from a Metadata Exchange (MEX) endpoint.</span></span>
