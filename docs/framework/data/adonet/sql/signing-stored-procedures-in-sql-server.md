# <a name="signing-stored-procedures-in-sql-server"></a><span data-ttu-id="d5706-102">Firmar procedimientos almacenados en SQL Server</span><span class="sxs-lookup"><span data-stu-id="d5706-102">Signing Stored Procedures in SQL Server</span></span>
<span data-ttu-id="d5706-103">Puede firmar un procedimiento almacenado con un certificado o una clave asimétrica.</span><span class="sxs-lookup"><span data-stu-id="d5706-103">You can sign a stored procedure with a certificate or an asymmetric key.</span></span> <span data-ttu-id="d5706-104">Esto está diseñado para los casos en los que no se pueden heredar permisos mediante encadenamiento de propiedad o cuando la cadena de propiedad está rota, como en SQL dinámico.</span><span class="sxs-lookup"><span data-stu-id="d5706-104">This is designed for scenarios when permissions cannot be inherited through ownership chaining or when the ownership chain is broken, such as dynamic SQL.</span></span> <span data-ttu-id="d5706-105">Entonces, se crea un usuario asignado al certificado, el cual concede permisos de usuario en los objetos a los que el procedimiento almacenado necesita tener acceso.</span><span class="sxs-lookup"><span data-stu-id="d5706-105">You then create a user mapped to the certificate, granting the certificate user permissions on the objects the stored procedure needs to access.</span></span>  
  
 <span data-ttu-id="d5706-106">Cuando se ejecuta el procedimiento almacenado, SQL Server combina los permisos del usuario del certificado con los del llamador.</span><span class="sxs-lookup"><span data-stu-id="d5706-106">When the stored procedure is executed, SQL Server combines the permissions of the certificate user with those of the caller.</span></span> <span data-ttu-id="d5706-107">A diferencia de la cláusula EXECUTE AS, aquí no cambia el contexto de ejecución del procedimiento.</span><span class="sxs-lookup"><span data-stu-id="d5706-107">Unlike the EXECUTE AS clause, it does not change the execution context of the procedure.</span></span> <span data-ttu-id="d5706-108">La funciones integradas que devuelven nombres de usuario e inicio de sesión devuelven el nombre del llamador, no el nombre de usuario del certificado.</span><span class="sxs-lookup"><span data-stu-id="d5706-108">Built-in functions that return login and user names return the name of the caller, not the certificate user name.</span></span>  
  
 <span data-ttu-id="d5706-109">Una firma digital es un resumen de datos cifrados con una clave privada del firmante.</span><span class="sxs-lookup"><span data-stu-id="d5706-109">A digital signature is a data digest encrypted with the private key of the signer.</span></span> <span data-ttu-id="d5706-110">La clave privada garantiza que la firma digital sea única para su portador o propietario.</span><span class="sxs-lookup"><span data-stu-id="d5706-110">The private key ensures that the digital signature is unique to its bearer or owner.</span></span> <span data-ttu-id="d5706-111">Se pueden firmar procedimientos almacenados, funciones o desencadenadores.</span><span class="sxs-lookup"><span data-stu-id="d5706-111">You can sign stored procedures, functions, or triggers.</span></span>  
  
> [!NOTE]
>  <span data-ttu-id="d5706-112">Se puede crear un certificado en la base de datos maestra para conceder permisos a nivel de servidor.</span><span class="sxs-lookup"><span data-stu-id="d5706-112">You can create a certificate in the master database to grant server-level permissions.</span></span>  
  
## <a name="creating-certificates"></a><span data-ttu-id="d5706-113">Crear certificados</span><span class="sxs-lookup"><span data-stu-id="d5706-113">Creating Certificates</span></span>  
 <span data-ttu-id="d5706-114">Al firmar un procedimiento almacenado con un certificado, se crea mediante la clave privada un resumen de datos que consta de hash cifrado de código del procedimiento almacenado.</span><span class="sxs-lookup"><span data-stu-id="d5706-114">When you sign a stored procedure with a certificate, a data digest consisting of the encrypted hash of the stored procedure code is created using the private key.</span></span> <span data-ttu-id="d5706-115">En tiempo de ejecución, el resumen de datos se descifra con la clave pública y se compara con el valor hash del procedimiento almacenado.</span><span class="sxs-lookup"><span data-stu-id="d5706-115">At run time, the data digest is decrypted with the public key and compared with the hash value of the stored procedure.</span></span> <span data-ttu-id="d5706-116">La modificación del procedimiento almacenado invalida el valor hash, de manera que la firma digital ya no coincide.</span><span class="sxs-lookup"><span data-stu-id="d5706-116">Modifying the stored procedure invalidates the hash value so that the digital signature no longer matches.</span></span> <span data-ttu-id="d5706-117">Con ello, se evita que alguien que no tenga acceso a la clave privada pueda cambiar el código del procedimiento almacenado.</span><span class="sxs-lookup"><span data-stu-id="d5706-117">This prevents someone who does not have access to the private key from changing the stored procedure code.</span></span> <span data-ttu-id="d5706-118">Por lo tanto, deberá volver a firmar el procedimiento almacenado cada vez que lo modifique.</span><span class="sxs-lookup"><span data-stu-id="d5706-118">Therefore you must re-sign the procedure each time you modify it.</span></span>  
  
 <span data-ttu-id="d5706-119">Son cuatro los pasos que implica la firma de un módulo:</span><span class="sxs-lookup"><span data-stu-id="d5706-119">There are four steps involved in signing a module:</span></span>

2.  <span data-ttu-id="d5706-123">Crear una base de datos asociada a un certificado utilizando la instrucción Transact-SQL `CREATE USER [userName] FROM CERTIFICATE [certificateName]`.</span><span class="sxs-lookup"><span data-stu-id="d5706-123">Create a database user associated with that certificate using the Transact-SQL `CREATE USER [userName] FROM CERTIFICATE [certificateName]` statement.</span></span> <span data-ttu-id="d5706-124">Este usuario existe únicamente en la base de datos y no está asociado a un inicio de sesión.</span><span class="sxs-lookup"><span data-stu-id="d5706-124">This user exists in the database only and is not associated with a login.</span></span>  
  
3.  <span data-ttu-id="d5706-125">Conceder al usuario del certificado los permisos necesarios en los objetos de la base de datos.</span><span class="sxs-lookup"><span data-stu-id="d5706-125">Grant the certificate user the required permissions on the database objects.</span></span>  
  
> [!NOTE]
>  <span data-ttu-id="d5706-126">Un certificado no puede conceder permisos a un usuario que tiene permisos revocados con la instrucción DENY.</span><span class="sxs-lookup"><span data-stu-id="d5706-126">A certificate cannot grant permissions to a user that has had permissions revoked using the DENY statement.</span></span> <span data-ttu-id="d5706-127">DENY siempre tiene prioridad sobre GRANT, lo que evita que el llamador herede permisos concedidos al usuario del certificado.</span><span class="sxs-lookup"><span data-stu-id="d5706-127">DENY always takes precedence over GRANT, preventing the caller from inheriting permissions granted to the certificate user.</span></span>  
  
1.  <span data-ttu-id="d5706-128">Firmar el procedimiento con el certificado utilizando la instrucción Transact-SQL `ADD SIGNATURE TO [procedureName] BY CERTIFICATE [certificateName]`.</span><span class="sxs-lookup"><span data-stu-id="d5706-128">Sign the procedure with the certificate using the Transact-SQL `ADD SIGNATURE TO [procedureName] BY CERTIFICATE [certificateName]` statement.</span></span>
