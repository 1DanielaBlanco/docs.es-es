# &#193;rbol de elementos y serializaci&#243;n
Los elementos de programación de WPF existen a menudo con alguna forma de relación recíproca en el árbol.  Por ejemplo, una interfaz de usuario de aplicación creada en XAML se puede conceptuar como un árbol de objetos.  El árbol de elementos puede estar dividido en dos árboles discretos pero, en ocasiones, paralelos: el [árbol lógico](GTMT) y el [árbol visual](GTMT).  La serialización en WPF implica guardar el estado de estos dos árboles así como el estado de la aplicación y escribirlo en un archivo, posiblemente en XAML.
