# Control avanzado de entrada manuscrita
[!INCLUDE[TLA2#tla_winclient](../../../../includes/tla2sharptla-winclient-md.md)] se entrega con <xref:System.Windows.Controls.InkCanvas>, que es un elemento que se puede incluir en una aplicación para empezar a recopilar y mostrar entradas de lápiz de forma inmediata.  Sin embargo, si el control <xref:System.Windows.Controls.InkCanvas> no proporciona un nivel de control suficiente, puede mantener el control en un nivel más alto mediante la personalización de sus propias clases de recopilación y representación de entrada de lápiz con <xref:System.Windows.Input.StylusPlugIns>.  
  
 Las clases de <xref:System.Windows.Input.StylusPlugIns> proporcionan un mecanismo para implementar un control de bajo nivel sobre la entrada <xref:System.Windows.Input.Stylus> y para representar la entrada de lápiz dinámicamente.  La clase <xref:System.Windows.Input.StylusPlugIns.StylusPlugIn> proporciona un mecanismo para implementar un comportamiento personalizado y aplicarlo a la secuencia de datos procedente del dispositivo de lápiz a fin de lograr un rendimiento óptimo.  <xref:System.Windows.Input.StylusPlugIns.DynamicRenderer>, un objeto <xref:System.Windows.Input.StylusPlugIns.StylusPlugIn> especializado, permite personalizar los datos de entrada de lápiz representados dinámicamente en tiempo real, lo que significa que <xref:System.Windows.Input.StylusPlugIns.DynamicRenderer> dibuja inmediatamente la entrada de lápiz digital mientras se generan los datos <xref:System.Windows.Input.StylusPoint>, por lo que parece "fluir" del dispositivo de lápiz.
