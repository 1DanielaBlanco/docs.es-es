# Instrucciones de nomenclatura
Un conjunto coherente de las convenciones de nomenclatura en el desarrollo de un marco de trabajo puede ser una contribución importante para facilidad de uso del marco de trabajo. Permite al marco que se utilizan muchos desarrolladores en proyectos ampliamente separados. Más allá de la coherencia del formulario, nombres de elementos de marco se deben entender fácilmente y deben transmitir la función de cada elemento.  
  
 El objetivo de este capítulo es proporcionar un conjunto coherente de las convenciones de nomenclatura que se crean nombres que tengan sentido inmediata a los desarrolladores.  
  
 Aunque adoptar estas convenciones de nomenclatura como instrucciones para el desarrollo general código daría como resultado una nomenclatura más coherente en todo el código, son necesarios sólo para aplicarlas a las API que se exponen públicamente \(tipos públicos o protegidos y los miembros y las interfaces implementadas de forma explícita\).
