# <a name="nested-types"></a><span data-ttu-id="73d4d-102">Tipos anidados</span><span class="sxs-lookup"><span data-stu-id="73d4d-102">Nested Types</span></span>
<span data-ttu-id="73d4d-103">Un tipo anidado es un tipo definido en el ámbito de otro tipo, que se denomina el tipo envolvente.</span><span class="sxs-lookup"><span data-stu-id="73d4d-103">A nested type is a type defined within the scope of another type, which is called the enclosing type.</span></span> <span data-ttu-id="73d4d-104">Un tipo anidado tiene acceso a todos los miembros de su tipo envolvente.</span><span class="sxs-lookup"><span data-stu-id="73d4d-104">A nested type has access to all members of its enclosing type.</span></span> <span data-ttu-id="73d4d-105">Por ejemplo, tiene acceso a los campos privados definidos en el tipo envolvente y proteger los campos definidos en todos los antecesores del tipo envolvente.</span><span class="sxs-lookup"><span data-stu-id="73d4d-105">For example, it has access to private fields defined in the enclosing type and to protected fields defined in all ascendants of the enclosing type.</span></span>  
  
 <span data-ttu-id="73d4d-106">En general, los tipos anidados deben usarse con moderación.</span><span class="sxs-lookup"><span data-stu-id="73d4d-106">In general, nested types should be used sparingly.</span></span> <span data-ttu-id="73d4d-107">Hay varias razones para ello.</span><span class="sxs-lookup"><span data-stu-id="73d4d-107">There are several reasons for this.</span></span> <span data-ttu-id="73d4d-108">Algunos desarrolladores no están totalmente familiarizados con el concepto.</span><span class="sxs-lookup"><span data-stu-id="73d4d-108">Some developers are not fully familiar with the concept.</span></span> <span data-ttu-id="73d4d-109">Por ejemplo, estos desarrolladores tenga problemas con la sintaxis de la declaración de variables de tipos anidados.</span><span class="sxs-lookup"><span data-stu-id="73d4d-109">These developers might, for example, have problems with the syntax of declaring variables of nested types.</span></span> <span data-ttu-id="73d4d-110">Los tipos anidados son también muy estrechamente con sus tipos envolventes y como tal no son adecuados para ser tipos de uso generales.</span><span class="sxs-lookup"><span data-stu-id="73d4d-110">Nested types are also very tightly coupled with their enclosing types, and as such are not suited to be general-purpose types.</span></span>  
  
 <span data-ttu-id="73d4d-111">Los tipos anidados son más adecuados para modelar los detalles de implementación de sus tipos envolventes.</span><span class="sxs-lookup"><span data-stu-id="73d4d-111">Nested types are best suited for modeling implementation details of their enclosing types.</span></span> <span data-ttu-id="73d4d-112">El usuario final rara vez deberían tener que declarar las variables de un tipo anidado y casi nunca debería tener que crear explícitamente instancias de tipos anidados.</span><span class="sxs-lookup"><span data-stu-id="73d4d-112">The end user should rarely have to declare variables of a nested type and almost never should have to explicitly instantiate nested types.</span></span> <span data-ttu-id="73d4d-113">Por ejemplo, el enumerador de una colección puede ser un tipo anidado de dicha recopilación.</span><span class="sxs-lookup"><span data-stu-id="73d4d-113">For example, the enumerator of a collection can be a nested type of that collection.</span></span> <span data-ttu-id="73d4d-114">Normalmente se crean instancias de los enumeradores por su tipo envolvente, y dado que muchos lenguajes admiten la instrucción foreach, variables de enumerador rara vez tienen que ser declarada por el usuario final.</span><span class="sxs-lookup"><span data-stu-id="73d4d-114">Enumerators are usually instantiated by their enclosing type, and because many languages support the foreach statement, enumerator variables rarely have to be declared by the end user.</span></span>  
  
 <span data-ttu-id="73d4d-115">**✓ DO** utilice tipos anidados cuando la relación entre el tipo anidado y su tipo exterior es tal que es deseable la semántica de accesibilidad de miembros.</span><span class="sxs-lookup"><span data-stu-id="73d4d-115">**✓ DO** use nested types when the relationship between the nested type and its outer type is such that member-accessibility semantics are desirable.</span></span>  
  
 <span data-ttu-id="73d4d-116">**X DO NOT** use los tipos anidados públicos como una agrupación lógica construir; usar espacios de nombres para este.</span><span class="sxs-lookup"><span data-stu-id="73d4d-116">**X DO NOT** use public nested types as a logical grouping construct; use namespaces for this.</span></span>  
  
 <span data-ttu-id="73d4d-117">**X AVOID** exponer públicamente los tipos anidados.</span><span class="sxs-lookup"><span data-stu-id="73d4d-117">**X AVOID** publicly exposed nested types.</span></span> <span data-ttu-id="73d4d-118">La única excepción a esto es si las variables del tipo anidado deben declararse sólo en escenarios poco habituales, como el uso de subclases u otros escenarios de personalización avanzada.</span><span class="sxs-lookup"><span data-stu-id="73d4d-118">The only exception to this is if variables of the nested type need to be declared only in rare scenarios such as subclassing or other advanced customization scenarios.</span></span>  
  
 <span data-ttu-id="73d4d-119">**X DO NOT** utilice tipos anidados si el tipo es probable que se hace referencia fuera del tipo contenedor.</span><span class="sxs-lookup"><span data-stu-id="73d4d-119">**X DO NOT** use nested types if the type is likely to be referenced outside of the containing type.</span></span>  
  
 <span data-ttu-id="73d4d-120">Por ejemplo, una enumeración que se pasa a un método definido en una clase no debe definirse como un tipo anidado en la clase.</span><span class="sxs-lookup"><span data-stu-id="73d4d-120">For example, an enum passed to a method defined on a class should not be defined as a nested type in the class.</span></span>  
  
 <span data-ttu-id="73d4d-121">**X DO NOT** utilice tipos anidados si necesitan que se creará una instancia de un código de cliente.</span><span class="sxs-lookup"><span data-stu-id="73d4d-121">**X DO NOT** use nested types if they need to be instantiated by client code.</span></span>  <span data-ttu-id="73d4d-122">Si un tipo tiene un constructor público, probablemente no debería anidarse.</span><span class="sxs-lookup"><span data-stu-id="73d4d-122">If a type has a public constructor, it should probably not be nested.</span></span>  
  
 <span data-ttu-id="73d4d-123">Si un tipo se puede crear instancias, que parece indicar el tipo tiene un lugar en el marco de trabajo por sí misma (puede crearla, trabajar con ellos y destruir sin necesidad de utilizar el tipo exterior) y, por tanto, no se pueden anidar.</span><span class="sxs-lookup"><span data-stu-id="73d4d-123">If a type can be instantiated, that seems to indicate the type has a place in the framework on its own (you can create it, work with it, and destroy it without ever using the outer type), and thus should not be nested.</span></span> <span data-ttu-id="73d4d-124">Tipos internos no se deberían reutilizar ampliamente fuera del tipo externo sin ninguna relación con el tipo exterior.</span><span class="sxs-lookup"><span data-stu-id="73d4d-124">Inner types should not be widely reused outside of the outer type without any relationship whatsoever to the outer type.</span></span>  
  
 <span data-ttu-id="73d4d-125">**X DO NOT** definir un tipo anidado como un miembro de una interfaz.</span><span class="sxs-lookup"><span data-stu-id="73d4d-125">**X DO NOT** define a nested type as a member of an interface.</span></span> <span data-ttu-id="73d4d-126">Muchos lenguajes no admiten este tipo de construcción.</span><span class="sxs-lookup"><span data-stu-id="73d4d-126">Many languages do not support such a construct.</span></span>
