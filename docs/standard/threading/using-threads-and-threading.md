# <a name="using-threads-and-threading"></a>Utilizar subprocesos y subprocesamiento
Los temas de esta sección explican la creación y administración de subprocesos administrados, cómo pasar datos a subprocesos administrados y recibir resultados devueltos y cómo destruir subprocesos y administrar una <xref:System.Threading.ThreadAbortException>.  
  
## <a name="in-this-section"></a>En esta sección  
 [Crear subprocesos y analizar los datos en el inicio](../../../docs/standard/threading/creating-threads-and-passing-data-at-start-time.md)  
 Explica y muestra la creación de subprocesos administrados, incluso cómo pasar datos a nuevos subprocesos y cómo recuperar los datos.  
  
 [Pausar y reanudar subprocesos](../../../docs/standard/threading/pausing-and-resuming-threads.md)  
 Explica las consecuencias de pausar y reanudar subprocesos administrados.  
  
 [Destruir subprocesos](../../../docs/standard/threading/destroying-threads.md)  
 Explica las consecuencias de destruir subprocesos administrados y cómo controlar un <xref:System.Threading.ThreadAbortException>.  
  
 [Planear subprocesos](../../../docs/standard/threading/scheduling-threads.md)  
 Explica las prioridades de subproceso y cómo afectan a la programación de subprocesos.  
  
## <a name="reference"></a>Referencia  
 <xref:System.Threading.Thread>  
 Proporciona documentación de referencia para el <xref:System.Threading.Thread> (clase), que representa un subproceso administrado, independientemente de que provenga de código no administrado o se creó en una aplicación administrada.  
  
 <xref:System.Threading.ThreadStart>  
 Proporciona documentación de referencia para el <xref:System.Threading.ThreadStart> delegado que representa los procedimientos de subproceso sin parámetros.  
  
 <xref:System.Threading.ParameterizedThreadStart>  
 Proporciona una manera sencilla para pasar datos a un procedimiento de subproceso, aunque sin establecimiento inflexible de tipos.
