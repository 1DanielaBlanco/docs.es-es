## <a name="handling-partial-failure"></a><span data-ttu-id="14b8e-114">Controlar errores parciales</span><span class="sxs-lookup"><span data-stu-id="14b8e-114">Handling partial failure</span></span>

<span data-ttu-id="14b8e-118">Dado que los clientes y servicios son procesos independientes, es posible que un servicio no pueda responder de manera oportuna a solicitud del cliente.</span><span class="sxs-lookup"><span data-stu-id="14b8e-118">Because clients and services are separate processes, a service might not be able to respond in a timely manner to a client's request.</span></span> <span data-ttu-id="14b8e-119">El servicio podría estar sobrecargado y responder con lentitud a las solicitudes, o podría no ser accesible durante un breve tiempo debido a problemas de red.</span><span class="sxs-lookup"><span data-stu-id="14b8e-119">The service might be overloaded and respond slowly to requests, or it might not be accessible for a short time because of network issues.</span></span>

<span data-ttu-id="14b8e-122">Una situación similar puede producirse en una aplicación que consuma servicios HTTP.</span><span class="sxs-lookup"><span data-stu-id="14b8e-122">A similar scenario might occur in an app that consumes HTTP services.</span></span> <span data-ttu-id="14b8e-123">La red o el propio servicio no estén disponible en la nube durante un error transitorio, corto.</span><span class="sxs-lookup"><span data-stu-id="14b8e-123">The network or the service itself might not be available in the cloud during a short, transient failure.</span></span>

<span data-ttu-id="14b8e-124">Una aplicación resistente como se muestra en la figura 4-9 debe implementar técnicas como "reintentos con retroceso exponencial" para dar una oportunidad para controlar errores transitorios en recursos de la aplicación.</span><span class="sxs-lookup"><span data-stu-id="14b8e-124">A resilient application like the one shown in Figure 4-9 should implement techniques like "retries with exponential backoff" to give the application an opportunity to handle transient failures in resources.</span></span> <span data-ttu-id="14b8e-125">También debe utilizar "los disyuntores" en sus aplicaciones.</span><span class="sxs-lookup"><span data-stu-id="14b8e-125">You also should use "circuit breakers" in your applications.</span></span> <span data-ttu-id="14b8e-126">Un disyuntor detiene una aplicación intente tener acceso a un recurso cuando es realmente un error de larga duración.</span><span class="sxs-lookup"><span data-stu-id="14b8e-126">A circuit breaker stops an application from trying to access a resource when it's actually a long-term failure.</span></span> <span data-ttu-id="14b8e-127">Mediante el uso de un disyuntor, la aplicación evita provocar una denegación de servicio a sí mismo.</span><span class="sxs-lookup"><span data-stu-id="14b8e-127">By using a circuit breaker, the application avoids provoking a denial of service to itself.</span></span>
