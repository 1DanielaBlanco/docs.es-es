## <a name="monolithic-application-deployed-as-a-container"></a><span data-ttu-id="55c3a-138">Aplicación monolítico implementada como un contenedor</span><span class="sxs-lookup"><span data-stu-id="55c3a-138">Monolithic application deployed as a container</span></span>

<span data-ttu-id="55c3a-139">Hay ventajas derivadas del uso de contenedores para administrar implementaciones monolíticas.</span><span class="sxs-lookup"><span data-stu-id="55c3a-139">There are benefits to using containers to manage monolithic deployments.</span></span> <span data-ttu-id="55c3a-140">Las instancias de contenedores de ajuste de escala es mucho más rápido y fácil de implementar máquinas virtuales adicionales.</span><span class="sxs-lookup"><span data-stu-id="55c3a-140">Scaling the instances of containers is far faster and easier than deploying additional VMs.</span></span> <span data-ttu-id="55c3a-141">Aunque los conjuntos de escalas de VM son una característica excelente para escalar las máquinas virtuales, que son necesarios para alojar los contenedores de Docker, tardan tiempo en configurar.</span><span class="sxs-lookup"><span data-stu-id="55c3a-141">Although VM Scale Sets are a great feature to scale VMs, which are required to host your Docker containers, they take time to set up.</span></span> <span data-ttu-id="55c3a-142">Cuando se implementa como instancias de aplicación, la configuración de la aplicación se administra como parte de la máquina virtual.</span><span class="sxs-lookup"><span data-stu-id="55c3a-142">When deployed as app instances, the configuration of the app is managed as part of the VM.</span></span>
