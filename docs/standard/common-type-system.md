*   <span data-ttu-id="2dd60-127">Estructuras</span><span class="sxs-lookup"><span data-stu-id="2dd60-127">Structures</span></span>
*   <span data-ttu-id="2dd60-128">Enumeraciones</span><span class="sxs-lookup"><span data-stu-id="2dd60-128">Enums</span></span>
*   <span data-ttu-id="2dd60-129">Interfaces</span><span class="sxs-lookup"><span data-stu-id="2dd60-129">Interfaces</span></span>
*   <span data-ttu-id="2dd60-130">Delegados</span><span class="sxs-lookup"><span data-stu-id="2dd60-130">Delegates</span></span>

## <a name="common-language-specification"></a><span data-ttu-id="2dd60-133">Common Language Specification</span><span class="sxs-lookup"><span data-stu-id="2dd60-133">Common Language Specification</span></span>

<span data-ttu-id="2dd60-134">Para habilitar escenarios de interoperabilidad completa, todos los objetos que se creen en el código deben basarse en algunas similitudes en los lenguajes que los usan (son sus _llamadores_).</span><span class="sxs-lookup"><span data-stu-id="2dd60-134">To enable full interoperability scenarios, all objects that are created in code must rely on some commonality in the languages that are consuming them (are their _callers_).</span></span> <span data-ttu-id="2dd60-135">Puesto que hay numerosos lenguajes diferentes, .NET ha especificado las similitudes en lo que se denomina **Common Language Specification** (CLS).</span><span class="sxs-lookup"><span data-stu-id="2dd60-135">Since there are numerous different languages, .NET has specified those commonalities in something called the **Common Language Specification** (CLS).</span></span> <span data-ttu-id="2dd60-136">CLS define un conjunto de características que son necesarias para muchas aplicaciones comunes.</span><span class="sxs-lookup"><span data-stu-id="2dd60-136">CLS defines a set of features that are needed by many common applications.</span></span> <span data-ttu-id="2dd60-137">También proporciona una especie de receta para cualquier lenguaje que se implementa sobre .NET sobre qué necesita para ser compatible.</span><span class="sxs-lookup"><span data-stu-id="2dd60-137">It also provides a sort of recipe for any language that is implemented on top of .NET on what it needs to support.</span></span>

<span data-ttu-id="2dd60-138">CLS es un subconjunto de CTS.</span><span class="sxs-lookup"><span data-stu-id="2dd60-138">CLS is a subset of the CTS.</span></span> <span data-ttu-id="2dd60-139">Esto significa que todas las reglas de CTS se aplican también a CLS, a menos que las reglas de CLS sean más estrictas.</span><span class="sxs-lookup"><span data-stu-id="2dd60-139">This means that all of the rules in the CTS also apply to the CLS, unless the CLS rules are more strict.</span></span> <span data-ttu-id="2dd60-140">Si un componente se compila solo con las reglas de CLS, es decir, expone solo las características de CLS en la API, se dice que es **conforme a CLS**.</span><span class="sxs-lookup"><span data-stu-id="2dd60-140">If a component is built using only the rules in the CLS, that is, it exposes only the CLS features in its API, it is said to be **CLS-compliant**.</span></span> <span data-ttu-id="2dd60-141">Por ejemplo, las `<framework-librares>` son conformes a CLS precisamente porque tienen que funcionar en todos los lenguajes que se admiten en .NET.</span><span class="sxs-lookup"><span data-stu-id="2dd60-141">For instance, the `<framework-librares>` are CLS-compliant precisely because they need to work across all of the languages that are supported on .NET.</span></span>
