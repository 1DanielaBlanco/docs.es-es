 <span data-ttu-id="041bf-107">El análisis se ve influido por las propiedades de un proveedor de formato que proporcione información, como las cadenas usadas para los separadores de fecha y hora, y los nombres de meses, días y eras.</span><span class="sxs-lookup"><span data-stu-id="041bf-107">Parsing is influenced by the properties of a format provider that supplies information such as the strings used for date and time separators, and the names of months, days, and eras.</span></span> <span data-ttu-id="041bf-108">El proveedor de formato es la actual <xref:System.Globalization.DateTimeFormatInfo> objeto, proporcionado implícitamente por la referencia cultural del subproceso actual o explícitamente por el <xref:System.IFormatProvider> parámetro de un método de análisis.</span><span class="sxs-lookup"><span data-stu-id="041bf-108">The format provider is the current <xref:System.Globalization.DateTimeFormatInfo> object, which is provided implicitly by the current thread culture or explicitly by the <xref:System.IFormatProvider> parameter of a parsing method.</span></span> <span data-ttu-id="041bf-109">Para el <xref:System.IFormatProvider> parámetro, especifique un <xref:System.Globalization.CultureInfo> objeto, que representa una referencia cultural, o un <xref:System.Globalization.DateTimeFormatInfo> objeto.</span><span class="sxs-lookup"><span data-stu-id="041bf-109">For the <xref:System.IFormatProvider> parameter, specify a <xref:System.Globalization.CultureInfo> object, which represents a culture, or a <xref:System.Globalization.DateTimeFormatInfo> object.</span></span>  
  
 <span data-ttu-id="041bf-110">La representación de cadena de una fecha que se vaya a analizar debe incluir el mes y, al menos, un día o año.</span><span class="sxs-lookup"><span data-stu-id="041bf-110">The string representation of a date to be parsed must include the month and at least a day or year.</span></span> <span data-ttu-id="041bf-111">La representación de cadena de una hora debe incluir la hora y, al menos, los minutos o el designador a. m./p. m.</span><span class="sxs-lookup"><span data-stu-id="041bf-111">The string representation of a time must include the hour and at least minutes or the AM/PM designator.</span></span> <span data-ttu-id="041bf-112">Pero, si es posible, el análisis proporciona valores predeterminados para los componentes omitidos.</span><span class="sxs-lookup"><span data-stu-id="041bf-112">However, parsing supplies default values for omitted components if possible.</span></span> <span data-ttu-id="041bf-113">Si falta una fecha, se usa como valor predeterminado la fecha actual; si falta un año, se usa como valor predeterminado el año actual; si falta un día del mes, se usa como valor predeterminado el primer día del mes; y si falta una hora, se usa como valor predeterminado la medianoche.</span><span class="sxs-lookup"><span data-stu-id="041bf-113">A missing date defaults to the current date, a missing year defaults to the current year, a missing day of the month defaults to the first day of the month, and a missing time defaults to midnight.</span></span>

 <span data-ttu-id="041bf-116">Además de un componente de fecha y hora, la representación de cadena de una fecha y hora puede incluir una diferencia horaria que indica cuánto difiere la hora respecto value adjusted to the local time zone of your machine.</span></span> <span data-ttu-id="041bf-120">Puede modificar este comportamiento mediante un <xref:System.Globalization.DateTimeStyles> constante con el método de análisis.</span><span class="sxs-lookup"><span data-stu-id="041bf-120">You can modify this behavior by using a <xref:System.Globalization.DateTimeStyles> constant with the parsing  <span data-ttu-id="041bf-121">El proveedor de formato también se usa para interpretar una fecha numérica ambigua.</span><span class="sxs-lookup"><span data-stu-id="041bf-121">The format provider is also used to interpret an ambiguous numeric date.</span></span> <span data-ttu-id="041bf-122">Por ejemplo, no está claro qué componentes de la fecha representada por la cadena "02/03/04" son el mes, el día y el año.</span><span class="sxs-lookup"><span data-stu-id="041bf-122">For example, it is not clear which components of the date represented by the string "02/03/04" are the month, day, and year.</span></span> <span data-ttu-id="041bf-123">En este caso, los componentes se interpretan según el orden de formatos de fecha similares del proveedor de formato.</span><span class="sxs-lookup"><span data-stu-id="041bf-123">In this case, the components are interpreted according to the order of similar date formats in the format provider.</span></span>
