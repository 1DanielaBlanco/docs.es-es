# <a name="microservices-architecture"></a><span data-ttu-id="9eb04-104">Arquitectura de microservicios</span><span class="sxs-lookup"><span data-stu-id="9eb04-104">Microservices architecture</span></span>

<span data-ttu-id="9eb04-105">Como su nombre indica, una arquitectura de microservicios es un enfoque para la generación de una aplicación de servidor como un conjunto de servicios pequeños.</span><span class="sxs-lookup"><span data-stu-id="9eb04-105">As the name implies, a microservices architecture is an approach to building a server application as a set of small services.</span></span> <span data-ttu-id="9eb04-106">Cada servicio se ejecuta en su propio proceso y se comunica con otros procesos mediante protocolos como HTTP/HTTPS, WebSockets, o [AMQP](https://en.wikipedia.org/wiki/Advanced_Message_Queuing_Protocol).</span><span class="sxs-lookup"><span data-stu-id="9eb04-106">Each service runs in its own process and communicates with other processes using protocols such as HTTP/HTTPS, WebSockets, or [AMQP](https://en.wikipedia.org/wiki/Advanced_Message_Queuing_Protocol).</span></span> <span data-ttu-id="9eb04-107">Cada microservicio implementa un dominio específico de end-to-end o una capacidad de negocio dentro de un determinado límite de contexto, y cada una se debe desarrollar de forma autónoma y se podrá implementar de forma independiente.</span><span class="sxs-lookup"><span data-stu-id="9eb04-107">Each microservice implements a specific end-to-end domain or business capability within a certain context boundary, and each must be developed autonomously and be deployable independently.</span></span> <span data-ttu-id="9eb04-108">Por último, cada microservicio debe poseer su modelo de datos de dominio relacionadas y la lógica del dominio (soberanía y administración de datos descentralizado) en función de las tecnologías de almacenamiento de datos diferentes (SQL, NoSQL) y lenguajes de programación diferentes.</span><span class="sxs-lookup"><span data-stu-id="9eb04-108">Finally, each microservice should own its related domain data model and domain logic (sovereignty and decentralized data management) based on different data storage technologies (SQL, NoSQL) and different programming languages.</span></span>

<span data-ttu-id="9eb04-109">¿Qué tamaño debe ser un microservicio?</span><span class="sxs-lookup"><span data-stu-id="9eb04-109">What size should a microservice be?</span></span> <span data-ttu-id="9eb04-110">Al desarrollar un microservicio, tamaño no debe ser importante.</span><span class="sxs-lookup"><span data-stu-id="9eb04-110">When developing a microservice, size should not be the important point.</span></span> <span data-ttu-id="9eb04-111">En su lugar, el punto importante que debe crear débilmente acoplados servicios por lo que tendrá la autonomía de desarrollo, implementación y escala, para cada servicio.</span><span class="sxs-lookup"><span data-stu-id="9eb04-111">Instead, the important point should be to create loosely coupled services so you have autonomy of development, deployment, and scale, for each service.</span></span> <span data-ttu-id="9eb04-112">Por supuesto, al identificar y diseñar microservicios, debe intentar que sean lo más pequeña posible siempre y cuando no tenga demasiadas dependencias directas con otros microservicios.</span><span class="sxs-lookup"><span data-stu-id="9eb04-112">Of course, when identifying and designing microservices, you should try to make them as small as possible as long as you do not have too many direct dependencies with other microservices.</span></span> <span data-ttu-id="9eb04-113">Más importante que el tamaño de la microservicio es su independencia de otros servicios y la cohesión interna debe tener.</span><span class="sxs-lookup"><span data-stu-id="9eb04-113">More important than the size of the microservice is the internal cohesion it must have and its independence from other services.</span></span>

<span data-ttu-id="9eb04-114">¿Por qué una arquitectura de microservicios?</span><span class="sxs-lookup"><span data-stu-id="9eb04-114">Why a microservices architecture?</span></span> <span data-ttu-id="9eb04-115">En resumen, proporciona agilidad a largo plazo.</span><span class="sxs-lookup"><span data-stu-id="9eb04-115">In short, it provides long-term agility.</span></span> <span data-ttu-id="9eb04-116">Microservicios habilita un mejor mantenimiento en sistemas altamente escalable, grandes y complejos, por lo que le permite crear aplicaciones basadas en muchos servicios pueden implementables de forma independiente que presentan los ciclos de vida granular y autónomos.</span><span class="sxs-lookup"><span data-stu-id="9eb04-116">Microservices enable better maintainability in complex, large, and highly-scalable systems by letting you create applications based on many independently deployable services that each have granular and autonomous lifecycles.</span></span>

<span data-ttu-id="9eb04-117">Como ventaja adicional, microservicios pueden escalar horizontalmente de forma independiente.</span><span class="sxs-lookup"><span data-stu-id="9eb04-117">As an additional benefit, microservices can scale out independently.</span></span> <span data-ttu-id="9eb04-118">En lugar de tener una sola aplicación monolítica que se debe escalar como una unidad, en su lugar, puede escalar horizontalmente microservicios específico.</span><span class="sxs-lookup"><span data-stu-id="9eb04-118">Instead of having a single monolithic application that you must scale out as a unit, you can instead scale out specific microservices.</span></span> <span data-ttu-id="9eb04-119">De este modo, puede escalar sólo el área funcional que necesita más procesamiento alimentación eléctrica o red de ancho de banda para admitir la demanda, en lugar de otras áreas de la aplicación que no es necesario escalar el escalado.</span><span class="sxs-lookup"><span data-stu-id="9eb04-119">That way, you can scale just the functional area that needs more processing power or network bandwidth to support demand, rather than scaling out other areas of the application that do not need to be scaled.</span></span> <span data-ttu-id="9eb04-120">Esto significa ahorros de costo porque necesita menos hardware.</span><span class="sxs-lookup"><span data-stu-id="9eb04-120">That means cost savings because you need less hardware.</span></span>

<span data-ttu-id="9eb04-124">Diseñar la arquitectura de integración continua de las aplicaciones específicas basadas en microservicios habilita y prácticas de la entrega continua.</span><span class="sxs-lookup"><span data-stu-id="9eb04-124">Architecting fine-grained microservices-based applications enables continuous integration and continuous delivery practices.</span></span> <span data-ttu-id="9eb04-125">También acelera la entrega de nuevas funciones en la aplicación.</span><span class="sxs-lookup"><span data-stu-id="9eb04-125">It also accelerates delivery of new functions into the application.</span></span> <span data-ttu-id="9eb04-126">Composición específica de las aplicaciones también le permite ejecutar y probar microservicios en aislamiento y les evolucionar de forma autónoma manteniendo contratos claros entre ellos.</span><span class="sxs-lookup"><span data-stu-id="9eb04-126">Fine-grained composition of applications also allows you to run and test microservices in isolation, and to evolve them autonomously while maintaining clear contracts between them.</span></span> <span data-ttu-id="9eb04-127">Siempre y cuando no cambie las interfaces o contratos, puede cambiar la implementación interna de cualquier microservicio o agregar nuevas funciones sin que ello interrumpa otros microservicios.</span><span class="sxs-lookup"><span data-stu-id="9eb04-127">As long as you do not change the interfaces or contracts, you can change the internal implementation of any microservice or add new functionality without breaking other microservices.</span></span>

<span data-ttu-id="9eb04-128">Éstos son aspectos importantes para habilitar el éxito de pasar a producción con un sistema basado en microservicios:</span><span class="sxs-lookup"><span data-stu-id="9eb04-128">The following are important aspects to enable success in going into production with a microservices-based system:</span></span>

-   <span data-ttu-id="9eb04-129">Comprobaciones de estado y la supervisión de los servicios y la infraestructura.</span><span class="sxs-lookup"><span data-stu-id="9eb04-129">Monitoring and health checks of the services and infrastructure.</span></span>

-   <span data-ttu-id="9eb04-130">Infraestructura escalable para los servicios (es decir, en la nube y orchestrators).</span><span class="sxs-lookup"><span data-stu-id="9eb04-130">Scalable infrastructure for the services (that is, cloud and orchestrators).</span></span>

-   <span data-ttu-id="9eb04-131">Diseño de seguridad e implementación en varios niveles: autenticación, autorización, administración de secretos, una comunicación segura, etcetera.</span><span class="sxs-lookup"><span data-stu-id="9eb04-131">Security design and implementation at multiple levels: authentication, authorization, secrets management, secure communication, etc.</span></span>

-   <span data-ttu-id="9eb04-132">Entrega rápido de aplicaciones, normalmente con los distintos equipos que centrarse en microservicios diferentes.</span><span class="sxs-lookup"><span data-stu-id="9eb04-132">Rapid application delivery, usually with different teams focusing on different microservices.</span></span>

-   <span data-ttu-id="9eb04-133">Infraestructura y las prácticas de DevOps y CI/CD.</span><span class="sxs-lookup"><span data-stu-id="9eb04-133">DevOps and CI/CD practices and infrastructure.</span></span>
