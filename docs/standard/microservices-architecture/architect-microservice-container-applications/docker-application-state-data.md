<span data-ttu-id="5c8e8-105">En la mayoría de los casos, un contenedor se puede considerar como una instancia de un proceso.</span><span class="sxs-lookup"><span data-stu-id="5c8e8-105">In most cases, you can think of a container as an instance of a process.</span></span> <span data-ttu-id="5c8e8-106">Un proceso no mantiene un estado persistente.</span><span class="sxs-lookup"><span data-stu-id="5c8e8-106">A process does not maintain persistent state.</span></span> <span data-ttu-id="5c8e8-107">Si bien un contenedor puede escribir en su almacenamiento local, suponer que una instancia permanecerá indefinidamente sería como suponer que una sola ubicación en memoria será duradera.</span><span class="sxs-lookup"><span data-stu-id="5c8e8-107">While a container can write to its local storage, assuming that an instance will be around indefinitely would be like assuming that a single location in memory will be durable.</span></span> <span data-ttu-id="5c8e8-108">Debe suponerse que las imágenes del contenedor, como los procesos, tienen varias instancias o que finalmente desaparecerán; si se administran con un orquestador de contenedores, se debe suponer que podrían desplazarse de un nodo o máquina virtual a otro.</span><span class="sxs-lookup"><span data-stu-id="5c8e8-108">Container images, like processes, should be assumed to have multiple instances or that they will eventually be killed; if they’re managed with a container orchestrator, it should be assumed that they might get moved from one node or VM to another.</span></span>

<span data-ttu-id="5c8e8-109">Docker ofrece una característica denominada el *sistema de archivos de superposición*.</span><span class="sxs-lookup"><span data-stu-id="5c8e8-109">Docker provides a feature named the *overlay file system*.</span></span> <span data-ttu-id="5c8e8-110">Esto implementa una tarea de copia en escritura que almacena información actualizada en el sistema de archivos raíz del contenedor.</span><span class="sxs-lookup"><span data-stu-id="5c8e8-110">This implements a copy-on-write task that stores updated information to the root file system of the container.</span></span> <span data-ttu-id="5c8e8-111">Esa información se combina con la imagen original en la que se basa el contenedor.</span><span class="sxs-lookup"><span data-stu-id="5c8e8-111">That information is in addition to the original image on which the container is based.</span></span> <span data-ttu-id="5c8e8-112">Si se elimina el contenedor del sistema, estos cambios se pierden.</span><span class="sxs-lookup"><span data-stu-id="5c8e8-112">If the container is deleted from the system, those changes are lost.</span></span> <span data-ttu-id="5c8e8-113">Por tanto, si bien es posible guardar el estado de un contenedor dentro de su almacenamiento local, diseñar un sistema sobre esta base entraría en conflicto con la idea del diseño del contenedor, que de manera predeterminada es sin estado.</span><span class="sxs-lookup"><span data-stu-id="5c8e8-113">Therefore, while it is possible to save the state of a container within its local storage, designing a system around this would conflict with the premise of container design, which by default is stateless.</span></span>

<span data-ttu-id="5c8e8-114">Las soluciones siguientes se usan para administrar datos persistentes en aplicaciones de Docker:</span><span class="sxs-lookup"><span data-stu-id="5c8e8-114">The following solutions are used to manage persistent data in Docker applications:</span></span>

-   <span data-ttu-id="5c8e8-116">[Contenedores de volúmenes de datos](https://docs.docker.com/engine/tutorials/dockervolumes/#creating-and-mounting-a-data-volume-container) que proporcionan almacenamiento compartido entre los contenedores mediante un contenedor externo.</span><span class="sxs-lookup"><span data-stu-id="5c8e8-116">[Data volume containers](https://docs.docker.com/engine/tutorials/dockervolumes/#creating-and-mounting-a-data-volume-container) that provide shared storage across containers using an external container.</span></span>

-   <span data-ttu-id="5c8e8-117">[Complementos de volumen](https://docs.docker.com/engine/tutorials/dockervolumes/) que montan volúmenes en los servicios remotos y proporcionan persistencia a largo plazo.</span><span class="sxs-lookup"><span data-stu-id="5c8e8-117">[Volume plugins](https://docs.docker.com/engine/tutorials/dockervolumes/) that mount volumes to remote services, providing long-term persistence.</span></span>

-   <span data-ttu-id="5c8e8-118">[Azure Storage](https://docs.microsoft.com/azure/storage/), que proporciona almacenamiento con distribución geográfica y representa una buena solución de persistencia a largo plazo para los contenedores.</span><span class="sxs-lookup"><span data-stu-id="5c8e8-118">[Azure Storage](https://docs.microsoft.com/azure/storage/), which provides geo-distributable storage, providing a good long-term persistence solution for containers.</span></span>

-   <span data-ttu-id="5c8e8-119">Bases de datos relacionales remotas como [Azure SQL Database](https://azure.microsoft.com/services/sql-database/), bases de datos NoSQL como [Azure Cosmos DB](https://docs.microsoft.com/azure/cosmos-db/introduction) o servicios de caché como [Redis](https://redis.io/).</span><span class="sxs-lookup"><span data-stu-id="5c8e8-119">Remote relational databases like [Azure SQL Database](https://azure.microsoft.com/services/sql-database/) or NoSQL databases like [Azure Cosmos DB](https://docs.microsoft.com/azure/cosmos-db/introduction), or cache services like [Redis](https://redis.io/).</span></span>

<span data-ttu-id="5c8e8-120">Los puntos siguientes proporcionan más información sobre estas opciones.</span><span class="sxs-lookup"><span data-stu-id="5c8e8-120">The following provides more detail about these options.</span></span>

<span data-ttu-id="5c8e8-121">Los **volúmenes de datos** son directorios asignados desde el sistema operativo del host a directorios en contenedores.</span><span class="sxs-lookup"><span <span data-ttu-id="5c8e8-127">Los **contenedores de volúmenes de datos** son una evolución de los volúmenes de datos normales.</span><span class="sxs-lookup"><span data-stu-id="5c8e8-127">**Data volume containers** are an evolution of regular data volumes.</span></span> <span data-ttu-id="5c8e8-128">Un contenedor de volúmenes de datos es un contenedor simple que contiene uno o más volúmenes de datos.</span><span class="sxs-lookup"><span data-stu-id="5c8e8-128">A data volume container is a simple container that has one or more data volumes within it.</span></span> <span data-ttu-id="5c8e8-129">El contenedor de volúmenes de datos proporciona acceso a los contenedores desde un punto de montaje central.</span><span class="sxs-lookup"><span data-stu-id="5c8e8-129">The data volume container provides access to containers from a central mount point.</span></span> <span data-ttu-id="5c8e8-130">Este método de acceso a datos es conveniente porque resume la ubicación de los datos originales.</span><span class="sxs-lookup"><span data-stu-id="5c8e8-130">This method of data access is convenient because it abstracts the location of the original data.</span></span> <span data-ttu-id="5c8e8-131">Aparte de eso, su comportamiento es similar al de un volumen de datos normal, por lo que los datos se guardan en este contenedor dedicado independientemente del ciclo de vida de los contenedores de la aplicación.</span><span class="sxs-lookup"><span data-stu-id="5c8e8-131">Other than that, its behavior is similar to that of a regular data volume, so data is persisted in this dedicated container independently of the lifecycle of the application’s containers.</span></span>
