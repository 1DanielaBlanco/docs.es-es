## <a name="the-specification-pattern"></a><span data-ttu-id="ffc3d-183">El patrón de especificación</span><span class="sxs-lookup"><span data-stu-id="ffc3d-183">The Specification pattern</span></span>

<span data-ttu-id="ffc3d-184">El patrón de especificación (el nombre completo sería patrón de especificación de consultas) es un modelo de diseño controlado por dominios diseñado como el lugar donde se puede incluir la definición de una consulta con lógica opcional de ordenación y paginación.</span><span class="sxs-lookup"><span data-stu-id="ffc3d-184">The Specification pattern (its full name would be Query-specification pattern) is a Domain-Driven Design pattern designed as the place where you can put the definition of a query with optional sorting and paging logic.</span></span>

<span data-ttu-id="ffc3d-185">El patrón de especificación define una consulta en un objeto.</span><span class="sxs-lookup"><span data-stu-id="ffc3d-185">The Specification pattern defines a query in an object.</span></span> <span data-ttu-id="ffc3d-186">Por ejemplo, para encapsular una consulta paginada que busca algunos productos, se puede crear una especificación PagedProduct que toma los parámetros de entrada necesarios (pageNumber, pageSize, filter, etc.).</span><span class="sxs-lookup"><span data-stu-id="ffc3d-186">For example, in order to encapsulate a paged query that searches for some products, you can create a PagedProduct specification that takes the necessary input parameters (pageNumber, pageSize, filter, etc.).</span></span> <span data-ttu-id="ffc3d-187">Después, dentro de cualquier método del repositorio (normalmente una sobrecarga de List()) aceptaría una ISpecification y ejecutaría la consulta esperada según esa especificación.</span><span class="sxs-lookup"><span data-stu-id="ffc3d-187">Then, within any Repository method (usually a List() overload) it would accept an ISpecification and run the expected query based on that specification.</span></span>

<span data-ttu-id="ffc3d-188">Este enfoque ofrece varias ventajas:</span><span class="sxs-lookup"><span data-stu-id="ffc3d-188">There are several benefits to this approach:</span></span>

* <span data-ttu-id="ffc3d-189">La especificación tiene un nombre (en contraposición a solo unas cuantas expresiones LINQ) del que se puede hablar.</span><span class="sxs-lookup"><span data-stu-id="ffc3d-189">The specification has a name (as opposed to just a bunch of LINQ expressions) that you can discuss about.</span></span>
