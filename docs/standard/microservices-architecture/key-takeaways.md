<span data-ttu-id="1f3a7-106">**Ventajas del uso de contenedores.**</span><span class="sxs-lookup"><span data-stu-id="1f3a7-106">**Benefits of using containers.**</span></span> <span data-ttu-id="1f3a7-107">Soluciones basadas en el contenedor proporcionan la ventaja importante de ahorro de costos porque los contenedores son una solución de problemas de implementación causados por la ausencia de dependencias en entornos de producción.</span><span class="sxs-lookup"><span data-stu-id="1f3a7-107">Container-based solutions provide the important benefit of cost savings because containers are a solution to deployment problems caused by the lack of dependencies in production environments.</span></span> <span data-ttu-id="1f3a7-108">Contenedores mejoran significativamente las operaciones de DevOps y producción.</span><span class="sxs-lookup"><span data-stu-id="1f3a7-108">Containers significantly improve DevOps and production operations.</span></span>

<span data-ttu-id="1f3a7-109">**Contenedores será ubicuos.**</span><span class="sxs-lookup"><span data-stu-id="1f3a7-109">**Containers will be ubiquitous.**</span></span> <span data-ttu-id="1f3a7-110">Contenedores de docker se está convirtiendo en el estándar de facto de la industria de contenedor, compatible con los proveedores más importantes en el ecosistema de Windows y Linux.</span><span class="sxs-lookup"><span data-stu-id="1f3a7-110">Docker-based containers are becoming the de facto standard in the container industry, supported by the most significant vendors in the Windows and Linux ecosystems.</span></span> <span data-ttu-id="1f3a7-111">Esto incluye Microsoft, Amazon AWS, Google y IBM.</span><span class="sxs-lookup"><span data-stu-id="1f3a7-111">This includes Microsoft, Amazon AWS, Google, and IBM.</span></span> <span data-ttu-id="1f3a7-112">En un futuro próximo, Docker probablemente será ubicuo en centros de datos en la nube y locales.</span><span class="sxs-lookup"><span data-stu-id="1f3a7-112">In the near future, Docker will probably be ubiquitous in both cloud and on-premises datacenters.</span></span>

<span data-ttu-id="1f3a7-113">**Contenedores como una unidad de implementación.**</span><span class="sxs-lookup"><span data-stu-id="1f3a7-113">**Containers as a unit of deployment.**</span></span> <span data-ttu-id="1f3a7-114">Un contenedor de Docker se está convirtiendo en la unidad de implementación para cualquier aplicación basada en el servidor o el servicio estándar.</span><span class="sxs-lookup"><span data-stu-id="1f3a7-114">A Docker container is becoming the standard unit of deployment for any server-based application or service.</span></span>

<span data-ttu-id="1f3a7-115">**Microservicios.**</span><span class="sxs-lookup"><span data-stu-id="1f3a7-115">**Microservices.**</span></span> <span data-ttu-id="1f3a7-116">La arquitectura de microservicios se está convirtiendo en el método preferido para las aplicaciones críticas distribuidas y grandes o complejas en función de varios subsistemas independientes en forma de servicios autónomos.</span><span class="sxs-lookup"><span data-stu-id="1f3a7-116">The microservices architecture is becoming the preferred approach for distributed and large or complex mission-critical applications based on multiple independent subsystems in the form of autonomous services.</span></span> <span data-ttu-id="1f3a7-117">En una arquitectura basada en microservicio, la aplicación se compila como una colección de servicios que pueden ser desarrollado, probado, control de versiones, implementar y escalar de forma independiente; Esto puede incluir cualquier base de datos autónomo relacionada.</span><span class="sxs-lookup"><span data-stu-id="1f3a7-117">In a microservice-based architecture, the application is built as a collection of services that can be developed, tested, versioned, deployed, and scaled independently; this can include any related autonomous database.</span></span>

<span data-ttu-id="1f3a7-118">**Diseño basado en dominio y SOA.**</span><span class="sxs-lookup"><span data-stu-id="1f3a7-118">**Domain-driven design and SOA.**</span></span> <span data-ttu-id="1f3a7-119">Los patrones de arquitectura de microservicios se derivan de la arquitectura orientada a servicios (SOA) y diseño basado en dominio (DDD).</span><span class="sxs-lookup"><span data-stu-id="1f3a7-119">The microservices architecture patterns derive from service-oriented architecture (SOA) and domain-driven design (DDD).</span></span> <span data-ttu-id="1f3a7-120">Al diseñar y desarrollar microservicios para entornos con evolucionando dar forma a un dominio en particular las reglas de negocios, es importante tener en cuenta DDD enfoques y patrones.</span><span class="sxs-lookup"><span data-stu-id="1f3a7-120">When you design and develop microservices for environments with evolving business rules shaping a particular domain, it is important to take into account DDD approaches and patterns.</span></span>

<span data-ttu-id="1f3a7-121">**Desafíos de Microservicios.**</span><span class="sxs-lookup"><span data-stu-id="1f3a7-121">**Microservices challenges.**</span></span> <span <span data-ttu-id="1f3a7-126">Estas soluciones incluyen aplicaciones grandes y complejas con varios subsistemas en constante evolución; en estos casos, merece la pena invertir en una arquitectura de <span data-ttu-id="1f3a7-127">**Contenedores para cualquier aplicación.**</span><span class="sxs-lookup"><span data-stu-id="1f3a7-127">**Containers for any application.**</span></span> <span data-ttu-id="1f3a7-128">Los contenedores son convenientes para microservicios, pero no son exclusivos para ellos.</span><span class="sxs-lookup"><span data-stu-id="1f3a7-128">Containers are convenient for microservices, but are not exclusive for them.</span></span> <span data-ttu-id="1f3a7-129">Contenedores también pueden usarse con aplicaciones monolíticas, incluidas las aplicaciones heredadas basado en .NET Framework tradicionales y y modernizado a través de contenedores de Windows.</span><span class="sxs-lookup"><span data-stu-id="1f3a7-129">Containers can also be used with monolithic applications, including legacy applications based on the traditional .NET Framework and modernized through Windows Containers.</span></span> <span data-ttu-id="1f3a7-130">Las ventajas de usar Docker, como muchos problemas de implementación de producción y proporcionar entornos de desarrollo y prueba del estado de la técnica, se aplican a muchos tipos diferentes de aplicaciones.</span><span class="sxs-lookup"><span data-stu-id="1f3a7-130">The benefits of using Docker, such as solving many deployment-to-production issues and providing state of the art Dev and Test environments, apply to many different types of applications.</span></span>

<span data-ttu-id="1f3a7-131">**CLI frente a IDE.**</span><span class="sxs-lookup"><span data-stu-id="1f3a7-131">**CLI versus IDE.**</span></span> <span data-ttu-id="1f3a7-132">Con herramientas de Microsoft, puede desarrollar aplicaciones .NET en contenedores con su método preferido.</span><span class="sxs-lookup"><span data-stu-id="1f3a7-132">With Microsoft tools, you can develop containerized .NET applications using your preferred approach.</span></span> <span data-ttu-id="1f3a7-133">Puede desarrollar con una CLI y un entorno basado en el editor mediante la CLI de Docker y el código de Visual Studio.</span><span class="sxs-lookup"><span data-stu-id="1f3a7-133">You can develop with a CLI and an editor-based environment by using the Docker CLI and Visual Studio Code.</span></span> <span data-ttu-id="1f3a7-134">O bien, puede usar un enfoque centrado en IDE con Visual Studio y sus características exclusivas de Docker, como como poder depurar aplicaciones de contenedor de varios.</span><span class="sxs-lookup"><span data-stu-id="1f3a7-134">Or you can use an IDE-focused approach with Visual Studio and its unique features for Docker, such as like being able to debug multi-container applications.</span></span>

<span data-ttu-id="1f3a7-135">**Aplicaciones de nube resistentes.**</span><span class="sxs-lookup"><span data-stu-id="1f3a7-135">**Resilient cloud applications.**</span></span> <span data-ttu-id="1f3a7-136">En sistemas basados en la nube y sistemas distribuidos en general, hay siempre el riesgo de error parcial.</span><span class="sxs-lookup"><span data-stu-id="1f3a7-136">In cloud-based systems and distributed systems in general, there is always the risk of partial failure.</span></span> <span data-ttu-id="1f3a7-137">Puesto que los clientes y servicios son procesos independientes (contenedores), es posible que un servicio no pueda responder de forma oportuna a solicitud del cliente.</span><span class="sxs-lookup"><span data-stu-id="1f3a7-137">Since clients and services are separate processes (containers), a service might not be able to respond in a timely way to a client’s request.</span></span> <span data-ttu-id="1f3a7-138">Por ejemplo, podría ser un servicio inactivo debido a un error parcial o mantenimiento; <span data-ttu-id="1f3a7-142">**Seguridad.**</span><span class="sxs-lookup"><span data-stu-id="1f3a7-142">**Security.**</span></span> <span data-ttu-id="1f3a7-143">Nuestro mundo moderno de contenedores y microservicios puede exponer nuevas vulnerabilidades.</span><span class="sxs-lookup"><span data-stu-id="1f3a7-143">Our modern world of containers and microservices can expose new vulnerabilities.</span></span> <span data-ttu-id="1f3a7-144">Seguridad de la aplicación básica se basa en la autenticación y autorización.
