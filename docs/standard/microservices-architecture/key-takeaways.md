<span data-ttu-id="0b0b2-106">**Ventajas del uso de contenedores.**</span><span class="sxs-lookup"><span data-stu-id="0b0b2-106">**Benefits of using containers.**</span></span> <span data-ttu-id="0b0b2-107">La gran ventaja que ofrecen las soluciones basadas en contenedor es el ahorro, porque con los contenedores se solucionan los problemas de implementación provocados por la falta de dependencias en entornos de producción.</span><span class="sxs-lookup"><span data-stu-id="0b0b2-107">Container-based solutions provide the important benefit of cost savings because containers are a solution to deployment problems caused by the lack of dependencies in production environments.</span></span> <span data-ttu-id="0b0b2-108">Los contenedores mejoran significativamente las operaciones de DevOps y producción.</span><span class="sxs-lookup"><span data-stu-id="0b0b2-108">Containers significantly improve DevOps and production operations.</span></span>

<span data-ttu-id="0b0b2-109">**El uso de los contenedores será generalizado.**</span><span class="sxs-lookup"><span data-stu-id="0b0b2-109">**Containers will be ubiquitous.**</span></span> <span data-ttu-id="0b0b2-110">Los contenedores basados en Docker se están convirtiendo en el estándar de facto del sector de los contenedores, ya que son compatibles con los proveedores más importantes de los ecosistemas de Windows y Linux.</span><span class="sxs-lookup"><span data-stu-id="0b0b2-110">Docker-based containers are becoming the de facto standard in the container industry, supported by the most significant vendors in the Windows and Linux ecosystems.</span></span> <span data-ttu-id="0b0b2-111">Esto incluye Microsoft, Amazon AWS, Google e IBM.</span><span class="sxs-lookup"><span data-stu-id="0b0b2-111">This includes Microsoft, Amazon AWS, Google, and IBM.</span></span> <span data-ttu-id="0b0b2-112">En un futuro próximo, el uso de Docker probablemente se ampliará a centros de datos en la nube y locales.</span><span class="sxs-lookup"><span data-stu-id="0b0b2-112">In the near future, Docker will probably be ubiquitous in both cloud and on-premises datacenters.</span></span>

<span data-ttu-id="0b0b2-113">**Los contenedores como una unidad de implementación.**</span><span class="sxs-lookup"><span data-stu-id="0b0b2-113">**Containers as a unit of deployment.**</span></span> <span data-ttu-id="0b0b2-114">Un contenedor de Docker se está convirtiendo en la unidad de implementación estándar para cualquier aplicación o servicio basados en servidor.</span><span class="sxs-lookup"><span data-stu-id="0b0b2-114">A Docker container is becoming the standard unit of deployment for any server-based application or service.</span></span>

<span data-ttu-id="0b0b2-115">**Microservicios.**</span><span class="sxs-lookup"><span data-stu-id="0b0b2-115">**Microservices.**</span></span> <span data-ttu-id="0b0b2-116">La arquitectura de microservicios se está convirtiendo en el método preferido para aplicaciones críticas distribuidas y grandes o complejas basadas en múltiples subsistemas independientes en forma de servicios autónomos.</span><span class="sxs-lookup"><span data-stu-id="0b0b2-116">The microservices architecture is becoming the preferred approach for distributed and large or complex mission-critical applications based on multiple independent subsystems in the form of autonomous services.</span></span> <span data-ttu-id="0b0b2-117">En una arquitectura basada en microservicios, la aplicación se crea como una colección de servicios que se pueden desarrollar, probar, versionear, implementar y escalar de forma independiente; esto puede incluir cualquier base de datos autónoma relacionada.</span><span class="sxs-lookup"><span data-stu-id="0b0b2-117">In a microservice-based architecture, the application is built as a collection of services that can be developed, tested, versioned, deployed, and scaled independently; this can include any related autonomous database.</span></span>

<span data-ttu-id="0b0b2-118">**Diseño guiado por el dominio y SOA.**</span><span class="sxs-lookup"><span data-stu-id="0b0b2-118">**Domain-driven design and SOA.**</span></span> <span data-ttu-id="0b0b2-119">Los patrones de arquitectura de microservicios se derivan de la arquitectura orientada a servicios (SOA) y del diseño guiado por el dominio (DDD).</span><span class="sxs-lookup"><span data-stu-id="0b0b2-119">The microservices architecture patterns derive from service-oriented architecture (SOA) and domain-driven design (DDD).</span></span> <span data-ttu-id="0b0b2-120">Al diseñar y desarrollar microservicios para entornos con reglas empresariales en evolución que dan forma a un dominio en particular, es importante tener en cuenta los enfoques y los patrones DDD.</span><span class="sxs-lookup"><span data-stu-id="0b0b2-120">When you design and develop microservices for environments with evolving business rules shaping a particular domain, it is important to take into account DDD approaches and patterns.</span></span>

<span data-ttu-id="0b0b2-121">**Retos de los microservicios.**</span><span class="sxs-lookup"><span data-stu-id="0b0b2-121">**Microservices challenges.**</span></span> <span data-ttu-id="0b0b2-124">Estos aspectos presentan un mayor nivel de complejidad que una aplicación monolítica tradicional.</span><span class="sxs-lookup"><span data-stu-id="0b0b2-124">These aspects <span data-ttu-id="0b0b2-126">Aquí se incluyen las aplicaciones grandes y complejas con varios subsistemas en constante evolución; en estos casos, merece la pena invertir en una arquitectura de software más compleja, ya que la agilidad y el mantenimiento de la aplicación serán mejores a largo plazo.</span><span class="sxs-lookup"><span data-stu-id="0b0b2-126">These include large and complex applications with multiple evolving subsystems; in these cases, it is worth investing in a more complex software architecture, because it will provide better long-term agility and application maintenance.</span></span>

<span data-ttu-id="0b0b2-127">**Contenedores para cualquier aplicación.**</span><span class="sxs-lookup"><span data-stu-id="0b0b2-127">**Containers for any application.**</span></span> <span data-ttu-id="0b0b2-128">Los contenedores son convenientes para los microservicios, pero no son exclusivos para ellos.</span><span class="sxs-lookup"><span data-stu-id="0b0b2-128">Containers are convenient for microservices, but are not exclusive for them.</span></span> <span data-ttu-id="0b0b2-129">Los contenedores también pueden usarse con aplicaciones monolíticas, incluidas las aplicaciones heredadas basadas en el .NET Framework tradicional y modernizadas a través de contenedores de Windows.</span><span class="sxs-lookup"><span data-stu-id="0b0b2-129">Containers can also be used with monolithic applications, including legacy applications based on the traditional .NET Framework and modernized through Windows Containers.</span></span> <span data-ttu-id="0b0b2-130">Las ventajas de usar Docker, como solucionar muchos problemas relacionados con el paso de la implementación a la producción y proporcionar entornos de desarrollo y prueba vanguardistas, se aplican a muchos tipos diferentes de aplicaciones.</span><span class="sxs-lookup"><span data-stu-id="0b0b2-130">The benefits of using Docker, such as solving many deployment-to-production issues and providing state of the art Dev and Test environments, apply to many different types of applications.</span></span>

<span data-ttu-id="0b0b2-131">**CLI frente a IDE.**</span><span class="sxs-lookup"><span data-stu-id="0b0b2-131">**CLI versus IDE.**</span></span> <span data-ttu-id="0b0b2-132">Con herramientas de Microsoft, puede desarrollar aplicaciones .NET en contenedores con su método preferido.</span><span class="sxs-lookup"><span data-stu-id="0b0b2-132">With Microsoft tools, you can develop containerized .NET applications using your preferred approach.</span></span> <span data-ttu-id="0b0b2-133">Puede desarrollar con una CLI y un entorno basado en editor mediante la CLI de Docker y Visual Studio Code.</span><span class="sxs-lookup"><span data-stu-id="0b0b2-133">You can develop with a CLI and an editor-based environment by using the Docker CLI and Visual Studio Code.</span></span> <span data-ttu-id="0b0b2-134">O bien, puede usar un enfoque centrado en IDE con Visual Studio y sus características exclusivas para Docker, como poder depurar aplicaciones de múltiples contenedores.</span><span class="sxs-lookup"><span data-stu-id="0b0b2-134">Or you can use an IDE-focused approach with Visual Studio and its unique features for Docker, such as like being able to debug multi-container applications.</span></span>

<span data-ttu-id="0b0b2-135">**Aplicaciones en la nube resistentes.**</span><span class="sxs-lookup"><span data-stu-id="0b0b2-135">**Resilient cloud applications.**</span></span> <span data-ttu-id="0b0b2-136">En los sistemas basados en la nube y en los sistemas distribuidos en general, siempre hay el riesgo de error parcial.</span><span class="sxs-lookup"><span data-stu-id="0b0b2-136">In cloud-based systems and distributed systems in general, there is always the risk of partial failure.</span></span> <span data-ttu-id="0b0b2-137">Puesto que los clientes y los servicios son procesos independientes (contenedores), es posible que un servicio no pueda responder de forma oportuna a la solicitud de un cliente.</span><span class="sxs-lookup"><span data-stu-id="0b0b2-137">Since clients and services are separate processes (containers), a service might not be able to respond in a timely way to a client’s request.</span></span> <span data-ttu-id="0b0b2-138">Por ejemplo, podría ser que un servicio estuviera inactivo a causa de un error parcial o por mantenimiento, que estuviera sobrecargado y respondiera muy lentamente a las solicitudes o bien que simplemente fuera inaccesible durante un breve tiempo debido a problemas de red.</span><span class="sxs-lookup"><span data-stu-id="0b0b2-138">For example, a service might be down because of a partial failure or for maintenance; the service might be overloaded and responding extremely slowly to requests; or it might simply not be accessible for a short time because of network issues.</span></span> <span data-ttu-id="0b0b2-139">Por tanto, una aplicación basada en la nube debe estar preparada para dichos errores y disponer de una estrategia para responder a los mismos.</span><span class="sxs-lookup"><span data-stu-id="0b0b2-139">Therefore, a cloud-based application must embrace those failures and have a strategy in place to respond to those failures.</span></span> <span data-ttu-id="0b0b2-140">Estas estrategias pueden incluir aplicar directivas de reintento (volver a enviar mensajes o solicitudes) e implementar patrones de interruptor para evitar una carga exponencial de solicitudes repetidas.</span><span class="sxs-lookup"><span data-stu-id="0b0b2-140">These strategies can include retry policies (resending messages or retrying requests) and implementing circuit-breaker patterns to avoid exponential load of repeated requests.</span></span> <span data-ttu-id="0b0b2-141">Básicamente, las aplicaciones basadas en la nube deben tener mecanismos resistentes, ya sean personalizados o basados en la infraestructura de nube, como los marcos de alto nivel de orquestadores o buses de servicio.</span><span class="sxs-lookup"><span data-stu-id="0b0b2-141">Basically, cloud-based applications must have resilient mechanisms—either custom ones, or ones based on cloud infrastructure, such as high-level frameworks from orchestrators or service buses.</span></span>

<span data-ttu-id="0b0b2-142">**Seguridad.**</span><span class="sxs-lookup"><span data-stu-id="0b0b2-142">**Security.**</span></span> <span data-ttu-id="0b0b2-143">Nuestro mundo moderno de contenedores y microservicios puede exponer nuevas vulnerabilidades.</span><span class="sxs-lookup"><span data-stu-id="0b0b2-143">Our modern world of containers and microservices can expose new vulnerabilities.</span></span> <span data-ttu-id="0b0b2-144">La seguridad de la aplicación básica se basa en la autenticación y la autorización;
