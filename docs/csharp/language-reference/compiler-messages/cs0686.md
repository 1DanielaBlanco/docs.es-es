 <span data-ttu-id="317e9-105">Sugerencia: este error se puede producir al implementar una interfaz que contiene nombres de métodos que están en conflicto con los métodos generados automáticamente y asociados a una propiedad o un evento.</span><span class="sxs-lookup"><span data-stu-id="317e9-105">Suggested: This error can occur when implementing an interface that contains method names which conflict with the auto-generated methods associated with a property or event.</span></span> <span data-ttu-id="317e9-106">Los métodos get/set correspondientes a las propiedades se generan como get_property y set_property, y los métodos add/remove correspondientes a los eventos se generan como add_event y remove_event.</span><span class="sxs-lookup"><span data-stu-id="317e9-106">The get/set methods for properties are generated as get_property and set_property, and the add/remove methods for events are generated as add_event and remove_event.</span></span> <span data-ttu-id="317e9-107">Si una interfaz contiene cualquiera de estos métodos, se produce un conflicto.</span><span class="sxs-lookup"><span data-stu-id="317e9-107">If an interface contains either of these methods, a conflict occurs.</span></span> <span data-ttu-id="317e9-108">Para evitar este error, implemente los métodos mediante una implementación de interfaces explícita.</span><span class="sxs-lookup"><span data-stu-id="317e9-108">To avoid this error, implement the methods using an explicit interface implementation.</span></span> <span data-ttu-id="317e9-109">Para ello, especifique la función de la siguiente manera:</span><span class="sxs-lookup"><span data-stu-id="317e9-109">To do this, specify the function as:</span></span>

  
## <a name="example"></a><span data-ttu-id="317e9-110">Ejemplo</span><span class="sxs-lookup"><span data-stu-id="317e9-110">Example</span></span>  
 <span data-ttu-id="317e9-111">En el ejemplo siguiente se genera la advertencia CS0686:</span><span class="sxs-lookup"><span data-stu-id="317e9-111">The following sample generates CS0686:</span></span>  
  
```csharp  
// CS0686.cs  
interface I  
{  
    int get_P();  
}  
  
class C : I  
{  
    public int P  
    {  
        get { return 1; }  // CS0686  
    }  
}  
// But the following is valid:  
class D : I  
{  
    int I.get_P() { return 1; }  
    public static void Main() {}  
}  
```  
  
## <a name="example"></a><span data-ttu-id="317e9-112">Ejemplo</span><span class="sxs-lookup"><span data-stu-id="317e9-112">Example</span></span>  
 <span data-ttu-id="317e9-113">Este error también se puede producir al declarar los eventos.</span><span class="sxs-lookup"><span data-stu-id="317e9-113">This error can also occur when declaring events.</span></span>  <span data-ttu-id="317e9-114">La construcción del evento genera automáticamente los métodos `add_``event` y `remove_``event`, que pueden estar en conflicto con los métodos del mismo nombre de una interfaz, como en el ejemplo siguiente:</span><span class="sxs-lookup"><span data-stu-id="317e9-114">The event construct automatically generates the `add_``event` and `remove_``event` methods, which could conflict with the methods of the same name in an interface, as in the following sample:</span></span>  
  
```csharp  
// CS0686b.cs  
using System;  
  
interface I  
{  
    void add_OnMyEvent(EventHandler e);  
}  
  
class C : I  
{  
    public event EventHandler OnMyEvent  
    {  
        add { }  // CS0686  
        remove { }  
    }  
}  
  
// Correct (using explicit interface implementation):  
class D : I  
{  
    void I.add_OnMyEvent(EventHandler e) {}  
    public static void Main() {}  
}  
```
