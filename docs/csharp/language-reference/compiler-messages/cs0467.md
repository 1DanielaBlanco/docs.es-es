# <a name="compiler-warning-level-2-cs0467"></a><span data-ttu-id="d26df-102">Advertencia del compilador (nivel 2) CS0467</span><span class="sxs-lookup"><span data-stu-id="d26df-102">Compiler Warning (level 2) CS0467</span></span>
<span data-ttu-id="d26df-103">Ambigüedad entre el método "method" y el no método "non-method". Se usará el grupo de métodos.</span><span class="sxs-lookup"><span data-stu-id="d26df-103">Ambiguity between method 'method' and non-method 'non-method'.Using method group.</span></span>  
  
 <span data-ttu-id="d26df-104">Los miembros heredados de distintas interfaces que tienen la misma firma producen un error de ambigüedad.</span><span class="sxs-lookup"><span data-stu-id="d26df-104">Inherited members from different interfaces that have the same signature  cause an ambiguity error.</span></span>  
  
## <a name="example"></a><span data-ttu-id="d26df-105">Ejemplo</span><span class="sxs-lookup"><span data-stu-id="d26df-105">Example</span></span>  
 <span data-ttu-id="d26df-106">En el ejemplo siguiente se genera el error CS0467.</span><span class="sxs-lookup"><span data-stu-id="d26df-106">The following example generates CS0467.</span></span>  
  
```csharp  
// CS0467.cs  
interface IList   
{  
    int Count { get; set; }  
}  
  
interface ICounter  
{  
    void Count(int i);  
}  
  
interface IListCounter : IList, ICounter {}  
  
class Driver   
{  
    void Test(IListCounter x)  
    {  
        // The following line causes the warning. The assignment also  
        // causes an error because you can't assign a value to a method.  
        x.Count = 1;  
        x.Count(3);     
        // To resolve the warning, you can change the name of the method or   
        // the property.  
  
        // You also can disambiguate by specifying IList or ICounter.  
        ((IList)x).Count = 1;  
        ((ICounter)x).Count(3);  
    }  
  
    static void Main()   
    {  
    }  
}  
```
