# <a name="compiler-error-cs0669"></a><span data-ttu-id="768bf-102">Error del compilador CS0669</span><span class="sxs-lookup"><span data-stu-id="768bf-102">Compiler Error CS0669</span></span>

<span data-ttu-id="768bf-103">Una clase con el atributo ComImport no puede tener un constructor definido por el usuario.</span><span class="sxs-lookup"><span data-stu-id="768bf-103">A class with the ComImport attribute cannot have a user-defined constructor.</span></span>

<span data-ttu-id="768bf-104">La capa de interoperabilidad COM en Common Language Runtime ofrece el constructor para las clases [ComImport](xref:System.Runtime.InteropServices.ComImportAttribute) .</span><span class="sxs-lookup"><span data-stu-id="768bf-104">The COM interop layer in the common language runtime supplies the constructor for [ComImport](xref:System.Runtime.InteropServices.ComImportAttribute) classes.</span></span> <span data-ttu-id="768bf-105">Por lo tanto, un objeto COM puede utilizarse como un objeto administrado en runtime.</span><span class="sxs-lookup"><span data-stu-id="768bf-105">Consequently, a COM object can be used as a managed object in the runtime.</span></span>

<span data-ttu-id="768bf-106">El ejemplo siguiente genera la advertencia CS0669:</span><span class="sxs-lookup"><span data-stu-id="768bf-106">The following sample generates CS0669:</span></span>

```csharp
// CS0669.cs
using System.Runtime.InteropServices;
[ComImport, Guid("00000000-0000-0000-0000-000000000001")]
class TestClass
{
   TestClass()   // CS0669, delete constructor to resolve
   {
   }

   public static void Main()
   {
   }
}
```
