## <a name="reference-types"></a><span data-ttu-id="2241c-113">Tipos de referencia</span><span class="sxs-lookup"><span data-stu-id="2241c-113">Reference types</span></span>  
<span data-ttu-id="2241c-114">Un tipo que se define como una [clase](../../../csharp/language-reference/keywords/class.md), es un *tipo de referencia*.</span><span class="sxs-lookup"><span data-stu-id="2241c-114">A type that is defined as a [class](../../../csharp/language-reference/keywords/class.md) is a *reference type*.</span></span> <span data-ttu-id="2241c-115">Al declarar una variable de un tipo de referencia en tiempo de ejecución, esta contendrá el valor [null](../../../csharp/language-reference/keywords/null.md) hasta que se cree expresamente una instancia de la clase mediante el operador [new](../../../csharp/language-reference/keywords/new.md) o que se le asigne un objeto creado en otro lugar, tal y como se muestra en el ejemplo siguiente:</span><span class="sxs-lookup"><span data-stu-id="2241c-115">At run time, when you declare a variable of a reference type, the variable contains the value [null](../../../csharp/language-reference/keywords/null.md) until you explicitly create an instance of the class by using the [new](../../../csharp/language-reference/keywords/new.md) operator, or assign it an object that has been created elsewhere, as shown in the following example:</span></span>

```csharp
MyClass mc = new MyClass();
MyClass mc2 = mc;
```

<span data-ttu-id="2241c-116">Cuando se crea el objeto, se asigna la memoria en el montón administrado y la variable solo contiene una referencia a la ubicación del objeto.</span><span class="sxs-lookup"><span data-stu-id="2241c-116">When the object is created, the memory is allocated on the managed heap, and the variable holds only a reference to the location of the object.</span></span> <span data-ttu-id="2241c-117">Los tipos del montón administrado producen sobrecarga cuando se asignan y cuando los reclama la función de administración de memoria automática de CLR, conocida como *recolección de elementos no utilizados*.</span><span class="sxs-lookup"><span data-stu-id="2241c-117">Types on the managed heap require overhead both when they are allocated and when they are reclaimed by the automatic memory management functionality of the CLR, which is known as *garbage collection*.</span></span> <span data-ttu-id="2241c-118">En cambio, la recolección de elementos no utilizados también está muy optimizada y no crea problemas de rendimiento en la mayoría de los escenarios.</span><span class="sxs-lookup"><span data-stu-id="2241c-118">However, garbage collection is also highly optimized, and in most scenarios, it does not create a performance issue.</span></span> <span data-ttu-id="2241c-119">Para obtener más información sobre la recolección de elementos no utilizados, vea [Administración automática de la memoria y recolección de elementos no utilizados](../../../standard/garbage-collection/gc.md).</span><span class="sxs-lookup"><span data-stu-id="2241c-119">For more information about garbage collection, see [Automatic memory management and garbage collection](../../../standard/garbage-collection/gc.md).</span></span>

## <a name="creating-objects"></a><span data-ttu-id="2241c-127">Crear objetos</span><span class="sxs-lookup"><span data-stu-id="2241c-127">Creating Objects</span></span>  
 <span data-ttu-id="2241c-128">Aunque a veces se usan indistintamente, una clase y un objeto son cosas diferentes.</span><span class="sxs-lookup"><span data-stu-id="2241c-128">Although they are sometimes used interchangeably, a class and an object are different things.</span></span> <span data-ttu-id="2241c-129">Una clase define un tipo de objeto, pero no es un objeto en sí.</span><span class="sxs-lookup"><span data-stu-id="2241c-129">A class defines a type of object, but it is not an object itself.</span></span> <span data-ttu-id="2241c-130">Un objeto es una entidad concreta basada en una clase y, a veces, se conoce como una instancia de una clase.</span><span class="sxs-lookup"><span data-stu-id="2241c-130">An object is a concrete entity based on a class, and is sometimes referred to as an instance of a class.</span></span>  
  
 <span data-ttu-id="2241c-131">Los objetos se pueden crear usando la palabra clave [new](../../../csharp/language-reference/keywords/new.md), seguida del nombre de la clase en la que se basará el objeto, como en este ejemplo:</span><span class="sxs-lookup"><span data-stu-id="2241c-131">Objects can be created by using the [new](../../../csharp/language-reference/keywords/new.md) keyword followed by the name of the class that the object will be based on, like this:</span></span>  

 ```csharp
 Customer object1 = new Customer();
 ```
  
 <span data-ttu-id="2241c-132">Cuando se crea una instancia de una clase, se vuelve a pasar al programador una referencia al objeto.</span><span class="sxs-lookup"><span data-stu-id="2241c-132">When an instance of a class is created, a reference to the object is passed back to the programmer.</span></span> <span data-ttu-id="2241c-133">En el ejemplo anterior, `object1` es una referencia a un objeto que se basa en `Customer`.</span><span class="sxs-lookup"><span data-stu-id="2241c-133">In the previous example, `object1` is a reference to an object that is based on `Customer`.</span></span> <span data-ttu-id="2241c-134">Esta referencia apunta al objeto nuevo, pero no contiene los datos del objeto.</span><span class="sxs-lookup"><span data-stu-id="2241c-134">This reference refers to the new object but does not contain the object data itself.</span></span> <span data-ttu-id="2241c-135">De hecho, puede crear una referencia de objeto sin tener que crear ningún objeto:</span><span class="sxs-lookup"><span data-stu-id="2241c-135">In fact, you can create an object reference without creating an object at all:</span></span>  
  
  ```csharp
  Customer object2;
  ```
  
 <span data-ttu-id="2241c-136">No se recomienda crear referencias de objeto como esta, que no hace referencia a ningún objeto, ya que, si se intenta obtener acceso a un objeto a través de este tipo de referencia, se producirá un error en tiempo de ejecución.</span><span class="sxs-lookup"><span data-stu-id="2241c-136">We don't recommend creating object references such as this one that don't refer to an object because trying to access an object through such a reference will fail at run time.</span></span> <span data-ttu-id="2241c-137">Pero dicha referencia puede haberse creado para hacer referencia a un objeto, ya sea creando un nuevo objeto o asignándola a un objeto existente, como en el siguiente ejemplo:</span><span class="sxs-lookup"><span data-stu-id="2241c-137">However, such a reference can be made to refer to an object, either by creating a new object, or by assigning it to an existing object, such as this:</span></span>  

 ```csharp
 Customer object3 = new Customer();
 Customer object4 = object3;
 ```
  
 <span data-ttu-id="2241c-138">Este código crea dos referencias de objeto que hacen referencia al mismo objeto.</span><span class="sxs-lookup"><span data-stu-id="2241c-138">This code creates two object references that both refer to the same object.</span></span> <span data-ttu-id="2241c-139">Por lo tanto, los cambios efectuados en el objeto mediante `object3` se reflejan en los usos posteriores de `object4`.</span><span class="sxs-lookup"><span data-stu-id="2241c-139">Therefore, any changes to the object made through `object3` are reflected in subsequent uses of `object4`.</span></span> <span data-ttu-id="2241c-140">Dado que los objetos basados en clases se tratan por referencia, las clases se denominan "tipos de referencia".</span><span class="sxs-lookup"><span data-stu-id="2241c-140">Because objects that are based on classes are referred to by reference, classes are known as reference types.</span></span>  
  
## <a name="class-inheritance"></a><span data-ttu-id="2241c-141">Herencia de clases</span><span class="sxs-lookup"><span data-stu-id="2241c-141">Class Inheritance</span></span>  

 <span data-ttu-id="2241c-142">Las clases admiten completamente la *herencia*, una característica fundamental de la programación orientada a objetos.</span><span class="sxs-lookup"><span data-stu-id="2241c-142">Classes fully support *inheritance*, a fundamental characteristic of object-oriented programming.</span></span> <span data-ttu-id="2241c-143">Al crear una clase, puede heredar de cualquier otra interfaz o clase que no esté definida como [sealed](../../../csharp/language-reference/keywords/sealed.md); y otras clases pueden heredar de su clase e invalidar los métodos virtuales de la clase.</span><span class="sxs-lookup"><span data-stu-id="2241c-143">When you create a class, you can inherit from any other interface or class that is not defined as [sealed](../../../csharp/language-reference/keywords/sealed.md), and other classes can inherit from your class and override class virtual methods.</span></span>

 <span data-ttu-id="2241c-144">La herencia se consigue mediante una *derivación*, en la que se declara una clase mediante una *clase base*, desde la que hereda los datos y el comportamiento.</span><span class="sxs-lookup"><span data-stu-id="2241c-144">Inheritance is accomplished by using a *derivation*, which means a class is declared by using a *base class* from which it inherits data and behavior.</span></span> <span data-ttu-id="2241c-145">Una clase base se especifica anexando dos puntos y el nombre de la clase base seguido del nombre de la clase derivada, como en el siguiente ejemplo:</span><span class="sxs-lookup"><span data-stu-id="2241c-145">A base class is specified by appending a colon and the name of the base class following the derived class name, like this:</span></span>

 ```csharp
     // Employee fields, properties, methods and events are inherited
     // New Manager fields, properties, methods and events go here...
 }
 ```

 <span data-ttu-id="2241c-146">Cuando una clase declara una clase base, hereda todos los miembros de la clase base excepto los constructores.</span><span class="sxs-lookup"><span data-stu-id="2241c-146">When a class declares a base class, it inherits all the members of the base class except the constructors.</span></span> <span data-ttu-id="2241c-147">Para obtener más información, vea [Herencia](../../../csharp/programming-guide/classes-and-structs/inheritance.md).</span><span class="sxs-lookup"><span data-stu-id="2241c-147">For more information, see [Inheritance](../../../csharp/programming-guide/classes-and-structs/inheritance.md).</span></span>

 <span data-ttu-id="2241c-152">Una clase puede declararse [abstracta](../../../csharp/language-reference/keywords/abstract.md).</span><span class="sxs-lookup"><span data-stu-id="2241c-152">A class can be declared [abstract](../../../csharp/language-reference/keywords/abstract.md).</span></span> <span data-ttu-id="2241c-153">Una clase abstracta contiene métodos abstractos que tienen una definición de firma, pero no tienen ninguna implementación.</span><span class="sxs-lookup"><span data-stu-id="2241c-153">An abstract class contains abstract methods that have a signature definition but no implementation.</span></span> <span data-ttu-id="2241c-154">No se pueden crear instancias de las clases abstractas.</span><span class="sxs-lookup"><span data-stu-id="2241c-154">Abstract classes cannot be instantiated.</span></span> <span data-ttu-id="2241c-155">Solo se pueden usar a través de las clases derivadas que implementan los métodos abstractos.</span><span class="sxs-lookup"><span data-stu-id="2241c-155">They can only be used through derived classes that implement the abstract methods.</span></span> <span data-ttu-id="2241c-156">Por el contrario, la clase [sealed](../../../csharp/language-reference/keywords/sealed.md) no permite que otras clases se deriven de ella.</span><span class="sxs-lookup"><span data-stu-id="2241c-156">By contrast, a [sealed](../../../csharp/language-reference/keywords/sealed.md) class does not allow other classes to derive from it.</span></span> <span data-ttu-id="2241c-157">Para más información, vea [Clases y miembros de clase abstractos y sellados](../../../csharp/programming-guide/classes-and-structs/abstract-and-sealed-classes-and-class-members.md).</span><span class="sxs-lookup"><span data-stu-id="2241c-157">For more information, see [Abstract and Sealed Classes and Class Members](../../../csharp/programming-guide/classes-and-structs/abstract-and-sealed-classes-and-class-members.md).</span></span>  
  
 <span data-ttu-id="2241c-158">Las definiciones de clase se pueden dividir entre distintos archivos de código fuente.</span><span class="sxs-lookup"><span data-stu-id="2241c-158">Class definitions can be split between different source files.</span></span> <span data-ttu-id="2241c-159">Para obtener más información, consulte [Clases y métodos parciales](../../../csharp/programming-guide/classes-and-structs/partial-classes-and-methods.md).</span><span class="sxs-lookup"><span data-stu-id="2241c-159">For more information, see [Partial Classes and Methods](../../../csharp/programming-guide/classes-and-structs/partial-classes-and-methods.md).</span></span>
