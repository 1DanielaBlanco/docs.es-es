# <a name="threading-c"></a><span data-ttu-id="c7a1b-102">Subprocesamiento (C#)</span><span class="sxs-lookup"><span data-stu-id="c7a1b-102">Threading (C#)</span></span>
<span data-ttu-id="c7a1b-103">El subprocesamiento permite a los programas de C# llevar a cabo un procesamiento simultáneo para que pueda hacer más de una operación a la vez.</span><span class="sxs-lookup"><span data-stu-id="c7a1b-103">Threading enables your C# program to perform concurrent processing so that you can do more than one operation at a time.</span></span> <span data-ttu-id="c7a1b-104">Por ejemplo, puede usar el subprocesamiento para hacer un seguimiento de los datos escritos por un usuario, realizar tareas en segundo plano y administrar transmisiones simultáneas de los datos escritos.</span><span class="sxs-lookup"><span data-stu-id="c7a1b-104">For example, you can use threading to monitor input from the user, perform background tasks, and handle simultaneous streams of input.</span></span>  
  
 <span data-ttu-id="c7a1b-105">Los subprocesos tienen las propiedades siguientes:</span><span class="sxs-lookup"><span data-stu-id="c7a1b-105">Threads have the following properties:</span></span>  
  
-   <span data-ttu-id="c7a1b-106">Permiten que un programa realice procesos simultáneos.</span><span class="sxs-lookup"><span data-stu-id="c7a1b-106">Threads enable your program to perform concurrent processing.</span></span>  
  
-   <span data-ttu-id="c7a1b-107">El espacio de nombres <xref:System.Threading> de .NET Framework permite usar subprocesos con más facilidad.</span><span class="sxs-lookup"><span data-stu-id="c7a1b-107">The .NET Framework <xref:System.Threading> namespace makes using threads easier.</span></span>  
  
-   <span data-ttu-id="c7a1b-108">Comparten los recursos de una aplicación.</span><span class="sxs-lookup"><span data-stu-id="c7a1b-108">Threads share the application's resources.</span></span> <span data-ttu-id="c7a1b-109">Para obtener más información, vea [Using Threads and Threading](../../../../../docs/standard/threading/using-threads-and-threading.md) (Uso de subprocesos y subprocesamiento).</span><span class="sxs-lookup"><span data-stu-id="c7a1b-109">For more information, see [Using Threads and Threading](../../../../../docs/standard/threading/using-threads-and-threading.md).</span></span>  
  
 <span data-ttu-id="c7a1b-110">De forma predeterminada, un programa de C# contiene un subproceso.</span><span class="sxs-lookup"><span data-stu-id="c7a1b-110">By default, a C# program has one thread.</span></span> <span data-ttu-id="c7a1b-111">Sin embargo, se pueden crear y usar subprocesos auxiliares para ejecutar código en paralelo con el subproceso principal.</span><span class="sxs-lookup"><span data-stu-id="c7a1b-111">However, auxiliary threads can be created and used to execute code in parallel with the primary thread.</span></span> <span data-ttu-id="c7a1b-112">Estos subprocesos se denominan a menudo *subprocesos de trabajo*.</span><span class="sxs-lookup"><span data-stu-id="c7a1b-112">These threads are often called *worker threads*.</span></span>  
  
 <span data-ttu-id="c7a1b-113">Los subprocesos de trabajo se pueden usar para realizar tareas urgentes o que requieren mucho tiempo sin limitar el subproceso principal.</span><span class="sxs-lookup"><span data-stu-id="c7a1b-113">Worker threads can be used to perform time-consuming or time-critical tasks without tying up the primary thread.</span></span> <span data-ttu-id="c7a1b-114">Por ejemplo, los subprocesos de trabajo se usan a menudo en aplicaciones de servidor para procesar las solicitudes entrantes sin tener que esperar a que se complete una solicitud anterior.</span><span class="sxs-lookup"><span data-stu-id="c7a1b-114">For example, worker threads are often used in server applications to fulfill incoming requests without waiting for the previous request to be completed.</span></span> <span data-ttu-id="c7a1b-115">También se suelen usar para realizar tareas “en segundo plano” en aplicaciones de escritorio para que el subproceso principal, que dirige los elementos de la interfaz de usuario, siga respondiendo a las acciones del usuario.</span><span class="sxs-lookup"><span data-stu-id="c7a1b-115">Worker threads are also used to perform "background" tasks in desktop applications so that the main thread--which drives user interface elements--remains responsive to user actions.</span></span>  
  
 <span data-ttu-id="c7a1b-116">El subprocesamiento resuelve los problemas de rendimiento y capacidad de respuesta, pero al hacerlo también puede crear problemas al compartir recursos, como interbloqueos y condiciones de carrera.</span><span class="sxs-lookup"><span data-stu-id="c7a1b-116">Threading solves problems with throughput and responsiveness, but it can also introduce resource-sharing issues such as deadlocks and race conditions.</span></span> <span data-ttu-id="c7a1b-117">Usar varios subprocesos es mejor en el caso de las tareas que requieren recursos distintos, como identificadores de archivos y conexiones de red.</span><span class="sxs-lookup"><span data-stu-id="c7a1b-117">Multiple threads are best for tasks that require different resources such as file handles and network connections.</span></span> <span data-ttu-id="c7a1b-118">Al asignar varios subprocesos a un único recurso, es probable que se produzcan problemas de sincronización, y el hecho de que los subprocesos se bloqueen a menudo al esperar por otros subprocesos iría en contra de la finalidad de usar varios subprocesos.</span><span class="sxs-lookup"><span data-stu-id="c7a1b-118">Assigning multiple threads to a single resource is likely to cause synchronization issues, and having threads frequently blocked when waiting for other threads defeats the purpose of using multiple threads.</span></span>  
  
 <span data-ttu-id="c7a1b-119">Una estrategia habitual es usar subprocesos de trabajo para llevar a cabo tareas urgentes o que requieren mucho tiempo pero no necesitan muchos de los recursos usados por otros subprocesos.</span><span class="sxs-lookup"><span data-stu-id="c7a1b-119">A common strategy is to use worker threads to perform time-consuming or time-critical tasks that do not require many of the resources used by other threads.</span></span> <span data-ttu-id="c7a1b-120">Naturalmente, hay varios subprocesos que deben acceder a algunos recursos de su programa.</span><span class="sxs-lookup"><span data-stu-id="c7a1b-120">Naturally, some resources in your program must be accessed by multiple threads.</span></span> <span data-ttu-id="c7a1b-121">Para estos casos, el espacio de nombres <xref:System.Threading> proporciona clases para sincronizar subprocesos.</span><span class="sxs-lookup"><span data-stu-id="c7a1b-121">For these cases, the <xref:System.Threading> namespace provides classes for synchronizing threads.</span></span> <span data-ttu-id="c7a1b-122">Entre estas clases se encuentran <xref:System.Threading.Mutex>, <xref:System.Threading.Monitor>, <xref:System.Threading.Interlocked>, <xref:System.Threading.AutoResetEvent> y <xref:System.Threading.ManualResetEvent>.</span><span class="sxs-lookup"><span data-stu-id="c7a1b-122">These classes include <xref:System.Threading.Mutex>, <xref:System.Threading.Monitor>, <xref:System.Threading.Interlocked>, <xref:System.Threading.AutoResetEvent>, and <xref:System.Threading.ManualResetEvent>.</span></span>  
  
 <span data-ttu-id="c7a1b-123">Puede usar todas estas clases o solo algunas de ellas para sincronizar las actividades de varios subprocesos, pero el lenguaje C# proporciona una parte de compatibilidad con los subprocesos.</span><span class="sxs-lookup"><span data-stu-id="c7a1b-123">You can use some or all these classes to synchronize the activities of multiple threads, but some support for threading is supported by the C# language.</span></span> <span data-ttu-id="c7a1b-124">Por ejemplo, una [instrucción lock](../../../../csharp/language-reference/keywords/lock-statement.md) proporciona características de sincronización mediante el uso implícito de <xref:System.Threading.Monitor>.</span><span class="sxs-lookup"><span data-stu-id="c7a1b-124">For example, the [Lock Statement](../../../../csharp/language-reference/keywords/lock-statement.md) provides synchronization features through implicit use of <xref:System.Threading.Monitor>.</span></span>
