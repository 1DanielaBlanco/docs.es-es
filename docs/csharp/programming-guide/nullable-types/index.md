# <a name="nullable-types-c-programming-guide"></a><span data-ttu-id="0cab3-103">Tipos que aceptan valores NULL (Guía de programación de C#)</span><span class="sxs-lookup"><span data-stu-id="0cab3-103">Nullable types (C# Programming Guide)</span></span>

<span data-ttu-id="0cab3-104">Los tipos que aceptan valores NULL son instancias de la estructura <xref:System.Nullable%601?displayProperty=nameWithType>.</span><span class="sxs-lookup"><span data-stu-id="0cab3-104">Nullable types are instances of the <xref:System.Nullable%601?displayProperty=nameWithType> struct.</span></span> <span data-ttu-id="0cab3-105">Los tipos que aceptan valores NULL pueden representar todos los valores de un tipo `T` subyacente y un valor [NULL](../../language-reference/keywords/null.md) adicional.</span><span class="sxs-lookup"><span data-stu-id="0cab3-105">Nullable types can represent all the values of an underlying type `T`, and an additional [null](../../language-reference/keywords/null.md) value.</span></span> <span data-ttu-id="0cab3-106">EL tipo `T` subyacente puede ser cualquier [tipo de valor](../../language-reference/keywords/value-types.md) que no acepte valores NULL.</span><span class="sxs-lookup"><span data-stu-id="0cab3-106">The underlying type `T` can be any non-nullable [value type](../../language-reference/keywords/value-types.md).</span></span> <span data-ttu-id="0cab3-107">`T` no puede ser un tipo de referencia.</span><span class="sxs-lookup"><span data-stu-id="0cab3-107">`T` cannot be a reference type.</span></span>

<span data-ttu-id="0cab3-108">Por ejemplo, puede asignar `null` o cualquier valor entero de <xref:System.Int32.MinValue?displayProperty=nameWithType> a <xref:System.Int32.MaxValue?displayProperty=nameWithType> a un elemento `Nullable<int>` y [true](../../language-reference/keywords/true-literal.md), [false](../../language-reference/keywords/false-literal.md) o `null` a un elemento `Nullable<bool>`.</span><span class="sxs-lookup"><span data-stu-id="0cab3-108">For example, you can assign `null` or any integer value from <xref:System.Int32.MinValue?displayProperty=nameWithType> to <xref:System.Int32.MaxValue?displayProperty=nameWithType> to a `Nullable<int>` and [true](../../language-reference/keywords/true-literal.md), [false](../../language-reference/keywords/false-literal.md), or `null` to a `Nullable<bool>`.</span></span>

<span data-ttu-id="0cab3-109">Los tipos que aceptan valores NULL se usan cuando es necesario representar el valor indefinido de un tipo subyacente.</span><span class="sxs-lookup"><span data-stu-id="0cab3-109">You use a nullable type when you need to represent the undefined value of an underlying type.</span></span> <span data-ttu-id="0cab3-110">Una variable booleana solo puede tener dos valores: true y false.</span><span class="sxs-lookup"><span data-stu-id="0cab3-110">A Boolean variable can have only two values: true and false.</span></span> <span data-ttu-id="0cab3-111">No hay ningún valor "sin definir".</span><span class="sxs-lookup"><span data-stu-id="0cab3-111">There is no "undefined" value.</span></span> <span data-ttu-id="0cab3-112">En muchas aplicaciones de programación, especialmente en las interacciones de las bases de datos, el valor de una variable puede ser indefinido o incluso no estar disponible.</span><span class="sxs-lookup"><span data-stu-id="0cab3-112">In many programming applications, most notably database interactions, a variable value can be undefined or missing.</span></span> <span data-ttu-id="0cab3-113">Por ejemplo, un campo de una base de datos puede contener los valores true o false, pero también puede no contener ningún valor.</span><span class="sxs-lookup"><span data-stu-id="0cab3-113">For example, a field in a database may contain the values true or false, or it may contain no value at all.</span></span> <span data-ttu-id="0cab3-114">En tal caso, se usa un tipo `Nullable<bool>`.</span><span class="sxs-lookup"><span data-stu-id="0cab3-114">You use a `Nullable<bool>` type in that case.</span></span>

<span data-ttu-id="0cab3-115">Los tipos que aceptan valores NULL tienen la siguientes características:</span><span class="sxs-lookup"><span data-stu-id="0cab3-115">Nullable types have the following characteristics:</span></span>
  
- <span data-ttu-id="0cab3-116">Los tipos que aceptan valores NULL representan variables de tipo de valor a las que se puede asignar el valor `null`.</span><span class="sxs-lookup"><span data-stu-id="0cab3-116">Nullable types represent value-type variables that can be assigned the `null` value.</span></span> <span data-ttu-id="0cab3-117">No se puede crear un tipo que acepta valores NULL basado en un tipo de referencia.</span><span class="sxs-lookup"><span data-stu-id="0cab3-117">You cannot create a nullable type based on a reference type.</span></span> <span data-ttu-id="0cab3-118">(Los tipos de referencia ya admiten el valor `null`).</span><span class="sxs-lookup"><span data-stu-id="0cab3-118">(Reference types already support the `null` value.)</span></span>  
  
- <span data-ttu-id="0cab3-119">La sintaxis `T?` es una abreviatura de `Nullable<T>`.</span><span class="sxs-lookup"><span data-stu-id="0cab3-119">The syntax `T?` is shorthand for `Nullable<T>`.</span></span> <span data-ttu-id="0cab3-120">Las dos formas son intercambiables.</span><span class="sxs-lookup"><span data-stu-id="0cab3-120">The two forms are interchangeable.</span></span>  
  
- <span data-ttu-id="0cab3-121">Asigne un valor a un tipo que acepta valores NULL del mismo modo que para un tipo de valor subyacente: `int? x = 10;` o `double? d = 4.108;`.</span><span class="sxs-lookup"><span data-stu-id="0cab3-121">Assign a value to a nullable type just as you would for an underlying value type: `int? x = 10;` or `double? d = 4.108;`.</span></span> <span data-ttu-id="0cab3-122">También puede asignar el valor `null`: `int? x = null;`.</span><span class="sxs-lookup"><span data-stu-id="0cab3-122">You also can assign the `null` value: `int? x = null;`.</span></span>  
  
- <span data-ttu-id="0cab3-123">Use las propiedades de solo lectura <xref:System.Nullable%601.HasValue%2A?displayProperty=nameWithType> y <xref:System.Nullable%601.Value%2A?displayProperty=nameWithType> para probar si hay valores NULL y recuperar el valor, como se muestra en el ejemplo siguiente: `if (x.HasValue) y = x.Value;`</span><span class="sxs-lookup"><span data-stu-id="0cab3-123">Use the <xref:System.Nullable%601.HasValue%2A?displayProperty=nameWithType> and <xref:System.Nullable%601.Value%2A?displayProperty=nameWithType> readonly properties to test for null and retrieve the value, as shown in the following example: `if (x.HasValue) y = x.Value;`</span></span>  
  
  - <span data-ttu-id="0cab3-124">La propiedad <xref:System.Nullable%601.HasValue%2A> devuelve `true` si la variable contiene un valor, o `false` si es `null`.</span><span class="sxs-lookup"><span data-stu-id="0cab3-124">The <xref:System.Nullable%601.HasValue%2A> property returns `true` if the variable contains a value, or `false` if it's `null`.</span></span>
  
  - <span data-ttu-id="0cab3-125">La propiedad <xref:System.Nullable%601.Value%2A> devuelve un valor si <xref:System.Nullable%601.HasValue%2A> devuelve `true`.</span><span class="sxs-lookup"><span data-stu-id="0cab3-125">The <xref:System.Nullable%601.Value%2A> property returns a value if <xref:System.Nullable%601.HasValue%2A> returns `true`.</span></span> <span data-ttu-id="0cab3-126">De lo contrario, se produce una excepción <xref:System.InvalidOperationException>.</span><span class="sxs-lookup"><span data-stu-id="0cab3-126">Otherwise, an <xref:System.InvalidOperationException> is thrown.</span></span>  
  
- <span data-ttu-id="0cab3-127">También puede utilizar los operadores `==` y `!=` con un tipo que acepta valores NULL, como se muestra en el ejemplo siguiente: `if (x != null) y = x.Value;`.</span><span class="sxs-lookup"><span data-stu-id="0cab3-127">You can also use the `==` and `!=` operators with a nullable type, as shown in the following example: `if (x != null) y = x.Value;`.</span></span> <span data-ttu-id="0cab3-128">Si `a` y `b` son ambos NULL, `a == b` se evalúa como `true`.</span><span class="sxs-lookup"><span data-stu-id="0cab3-128">If `a` and `b` are both null, `a == b` evaluates to `true`.</span></span>  

- <span data-ttu-id="0cab3-129">A partir de C# 7.0, se puede usar la [coincidencia de patrones](../../pattern-matching.md#the-is-type-pattern-expression) tanto para examinar como para obtener un valor de un tipo que acepta valores NULL: `if (x is int valueOfX) y = valueOfX;`.</span><span class="sxs-lookup"><span data-stu-id="0cab3-129">Beginning with C# 7.0, you can use [pattern matching](../../pattern-matching.md#the-is-type-pattern-expression) to both examine and get a value of a nullable type: `if (x is int valueOfX) y = valueOfX;`.</span></span>
  
- <span data-ttu-id="0cab3-130">El valor predeterminado de `T?` es una instancia cuya propiedad <xref:System.Nullable%601.HasValue%2A> devuelve `false`.</span><span class="sxs-lookup"><span data-stu-id="0cab3-130">The default value of `T?` is an instance whose <xref:System.Nullable%601.HasValue%2A> property returns `false`.</span></span>  

- <span data-ttu-id="0cab3-131">Use el método <xref:System.Nullable%601.GetValueOrDefault> para devolver el valor asignado, o bien el [valor predeterminado](../../language-reference/keywords/default-values-table.md) del tipo subyacente si el valor es `null`.</span><span class="sxs-lookup"><span data-stu-id="0cab3-131">Use the <xref:System.Nullable%601.GetValueOrDefault> method to return either the assigned value, or the [default](../../language-reference/keywords/default-values-table.md) value of the underlying value type if the value of the nullable type is `null`.</span></span>  

- <span data-ttu-id="0cab3-132">Use el método <xref:System.Nullable%601.GetValueOrDefault(%600)> para devolver el valor asignado, o bien el valor predeterminado proporcionado si el valor es `null`.</span><span class="sxs-lookup"><span data-stu-id="0cab3-132">Use the <xref:System.Nullable%601.GetValueOrDefault(%600)> method to return either the assigned value, or the provided default value if the value of the nullable type is `null`.</span></span>
  
- <span data-ttu-id="0cab3-133">Use el [operador de fusión NULL](../../language-reference/operators/null-coalescing-operator.md) (`??`) para asignar un valor a un tipo subyacente basado en un valor del tipo que acepta valores NULL: `int? x = null; int y = x ?? -1;`.</span><span class="sxs-lookup"><span data-stu-id="0cab3-133">Use the [null-coalescing operator](../../language-reference/operators/null-coalescing-operator.md), `??`, to assign a value to an underlying type based on a value of the nullable type: `int? x = null; int y = x ?? -1;`.</span></span> <span data-ttu-id="0cab3-134">En el ejemplo, dado que `x` es NULL, el valor resultante de `y` es `-1`.</span><span class="sxs-lookup"><span data-stu-id="0cab3-134">In the example, since `x` is null, the result value of `y` is `-1`.</span></span>

- <span data-ttu-id="0cab3-135">Si se establece una conversión definida por el usuario entre dos tipos de datos, esa misma conversión también se puede usar con las versiones que aceptan valores NULL de dichos tipos de datos.</span><span class="sxs-lookup"><span data-stu-id="0cab3-135">If a user-defined conversion is defined between two data types, the same conversion can also be used with the nullable versions of these data types.</span></span>
  
- <span data-ttu-id="0cab3-136">No se permiten los tipos anidados que aceptan valores NULL.</span><span class="sxs-lookup"><span data-stu-id="0cab3-136">Nested nullable types are not allowed.</span></span> <span data-ttu-id="0cab3-137">La línea siguiente no se compilará: `Nullable<Nullable<int>> n;`</span><span class="sxs-lookup"><span data-stu-id="0cab3-137">The following line doesn't compile: `Nullable<Nullable<int>> n;`</span></span>
